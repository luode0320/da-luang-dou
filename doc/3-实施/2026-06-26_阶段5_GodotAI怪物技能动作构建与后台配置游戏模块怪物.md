# 阶段 5：Godot AI MCP 怪物 + 技能 + 动作构建 + 后台配置游戏模块怪物

## 阶段目标

阶段 5 负责怪物、精英怪和 Boss。怪物设计必须服务模块关卡难度，不通过单纯堆血量控制难度。

长期投射物、命中反馈、Buff / Debuff 效果包和技能表现载体的正文契约，统一以 `doc/1-架构/附录-武器投射物与命中效果规范.md` 与 `doc/1-架构/附录-技能配置与战斗表现规范.md` 为准；本文只保留阶段实施范围。

正式怪物素材必须先通过 `imagegen` 设计，再通过 Godot AI MCP 接入动作、技能、Boss 形态和特效。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 普通怪物 | 只有 3 个基础技能。 |
| 精英怪物 | 3 个基础技能 + 1 个终极技能。 |
| Boss 基础形态 | 3 个基础技能 + 1 个终极技能。 |
| Boss 终极形态 | 基础形态能力外，再加 3 个基础技能 + 1 个终极技能。 |
| Boss 狂暴形态 | 终极形态死亡后按难度概率触发，恢复满血，攻速、移速、技能 CD 加快一倍，血量上限不增加。 |
| 动作动画 | 待机、移动、攻击、施法、受击、死亡、Boss 变身。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 怪物基础信息 | ID、中文名、类型、所属模块、标签。 |
| 技能组 | 基础技能、终极技能、阶段技能。 |
| 难度映射 | 普通、噩梦、地狱的技能开放和 Boss 出现规则。 |
| Boss 形态 | 基础形态、终极形态、狂暴形态、触发概率和倍率。 |
| 掉落配置 | 金币掉落、道具池引用、Boss 奖励池。 |
| 素材绑定 | 动作帧、特效、音效、投射物。 |

## 配置目录建议

```text
game/data/modules/<module_id>/enemies/<enemy_id>/
├── config.yaml                                # 怪物主配置
├── assets/                                    # 怪物动作帧
├── skills/                                    # 怪物技能配置
├── vfx/                                       # 技能特效
└── overrides/                                 # 模块局部覆盖

game/data/modules/<module_id>/bosses/<boss_id>/
├── config.yaml                                # Boss 主配置
├── assets/                                    # Boss 动作与形态素材
├── phases/                                    # 基础、终极、狂暴形态配置
├── skills/                                    # Boss 技能配置
└── overrides/
```

## 怪物字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 怪物稳定 ID。 |
| `type` | 普通、精英、BOSS。 |
| `hp` | 血量，不随难度堆叠。 |
| `move_speed` | 移速。 |
| `attack_speed` | 攻速。 |
| `base_skill_ids` | 基础技能列表。 |
| `ultimate_skill_ids` | 终极技能列表。 |
| `phase_ids` | Boss 阶段列表。 |
| `difficulty_unlock_map` | 难度技能开放映射。 |
| `loot_table_id` | 掉落表。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-5-001 | 普通怪、精英怪、Boss 技能数量符合分层规则。 |
| AC-5-002 | 普通、噩梦、地狱难度通过配置控制技能开放和 Boss 形态。 |
| AC-5-003 | 同类怪物不同难度血量一致。 |
| AC-5-004 | Boss 狂暴形态按概率触发，不是每次必触发。 |
| AC-5-005 | 怪物动作和技能素材先经 `imagegen` 设计，再由 Godot AI MCP 接入。 |
