# 玩家设置与运行时操作规范

## 目标

本附录用于把以下长期规则收口为单一口径：

- 默认设置如何由后台与发布配置定义
- 玩家个人设置如何与发布默认值分层
- 语言、音量、键位、分辨率、无障碍如何进入运行时
- 暂停与加速如何成为统一运行时操作规则
- 什么属于内容发布配置，什么属于玩家本地偏好

它解决的问题不是“首页有个设置按钮”，而是保证后续首页、关卡、打包和验收都围绕同一套设置与操作边界推进。

- 首页设置入口

- PC 桌面体验相关基础操作

## 一、分层原则

### 1. 默认设置与玩家个人设置必须分层

- 默认设置由后台配置并进入发布包。
- 玩家个人设置由运行时本地保存。
- 玩家个人设置可以覆盖默认值，但不能反向改写发布配置。
- 设置入口如何进入、默认设置何时读入、个人偏好何时合并和何时在运行态继续生效，继续转交 [8-业务链路-首页设置与运行时操作](8-业务链路-首页设置与运行时操作.md)。

### 2. 设置项分两类

| 类型 | 说明 | 真源 |
| --- | --- | --- |
| 发布默认设置 | 语言默认值、可选分辨率、默认音量档位、默认键位方案、可见的无障碍选项 | 发布配置 |
| 玩家个人设置 | 玩家当前分辨率、当前音量、当前键位、当前无障碍选择、当前加速偏好 | 本地设置存储 |

### 3. 运行时操作必须统一口径

以下操作不应分散在不同系统各自解释：

- `I` 键背包暂停
- 1x / 2x / 3x 加速
- 键位映射
- 音量分组
- 语言切换
- 无障碍开关

本附录只定义这些操作的统一规则口径，不负责逐步执行顺序。

## 二、最小设置集合

### 2.1 PC 基础设置

长期最小集合至少包含：

- 窗口模式
- 分辨率
- 全屏 / 窗口化切换
- 主音量、音乐音量、音效音量、UI 音量
- 语言
- 键位映射
- 加速档位
- 无障碍基础选项

### 2.2 无障碍最小集合

至少包含：

- 字体大小
- UI 缩放
- 高对比度或色弱模式
- 字幕 / 提示停留时长
- 输入提示显示方式

### 2.3 第一版边界

- 第一版可以只做基础展示与基础可用，不必一次做完整高级面板。
- 但字段口径、默认值、运行时读取路径和验证口径必须先统一。

## 三、推荐配置载体

### 3.1 发布默认设置

建议默认设置配置进入 `control/` 与全局配置命名空间，至少包含：

- `settings/defaults.yaml`
- `settings/options.yaml`
- `settings/accessibility.yaml`
- `settings/input_profiles.yaml`

这些文件属于发布配置的一部分，用于告诉游戏“默认应该怎么表现、允许玩家选什么”。

它们不负责说明：

- 发布后由哪条主链路读入
- 玩家从首页进入设置后的逐步流程
- 运行时在不同场景下先应用哪一组设置

### 3.2 玩家个人设置

玩家个人设置不进入发布包，长期建议单独保存在本地设置载体中，至少记录：

- `language`
- `window_mode`
- `resolution`
- `master_volume`
- `music_volume`
- `sfx_volume`
- `ui_volume`
- `key_bindings`
- `accessibility_flags`
- `preferred_speed_level`

## 四、运行时合并规则

统一生效顺序建议为：

`玩家个人设置 > 发布默认设置 > 全局默认值 > 代码内置默认值`

统一约束：

1. 发布默认设置决定“默认值和允许选项”。
2. 玩家个人设置决定“本机当前实际选择”。
3. 若个人设置缺失或损坏，回退到发布默认设置。
4. 若发布默认设置缺失，再回退到全局或内置默认值，但必须记录警告。

本附录只固定优先级，不代写“启动后、进入设置后、进入关卡后”的具体时序。

## 五、暂停与加速规则

### 5.1 暂停规则

- `I` 键打开背包时进入暂停态。
- 暂停态下怪物技能、角色技能、关卡时间和相关动作推进统一停止。
- 暂停态允许查看背包、查看提示、进行局内抽道具交互。

### 5.2 加速规则

- 第一版固定支持 1x、2x、3x 三档。
- 加速倍率必须统一作用于：
  - 关卡时间
  - 冷却
  - 刷怪节奏
  - 掉落节奏
  - 存档统计口径

### 5.3 阻断条件

以下情况默认阻断：

- 暂停后仍推进战斗时间
- 加速只影响部分系统，导致不同步
- 键位变更导致核心入口不可用且无回退

## 六、后台与游戏的职责边界

### 6.1 后台负责

- 配置默认设置
- 配置允许选项集合
- 配置语言与无障碍默认项
- 配置首页设置入口是否显示

### 6.2 游戏负责

- 读取默认设置
- 读取并持久化玩家个人设置
- 合并默认设置与玩家偏好
- 在运行时统一执行暂停、加速和输入映射

### 6.3 不应混淆

- 不把玩家个人音量、键位或分辨率写回后台发布配置
- 不把后台草稿当成运行时设置真源
- 不把暂停或加速逻辑写成局部特判

## 七、最小验证口径

阶段验证至少应覆盖：

1. 首页能进入设置入口
2. 默认设置来自发布配置，而不是写死
3. 个人设置重启后可继续生效
4. `I` 键背包暂停能停止怪物与角色技能动作
5. 1x / 2x / 3x 加速对公共时间和冷却保持一致
6. 音量、分辨率、语言、键位至少可被读取并应用

## 九、结论

玩家设置与运行时操作真正要先写硬的是这条边界：

- 后台定义默认值
- 玩家保存个人偏好
- 运行时统一合并并执行
- 暂停与加速按统一规则作用于全局

只要这条边界先稳定，后续首页设置、PC 基础体验、国际化预留、无障碍支持和运行时操作一致性才能沿着同一套长期架构推进。
