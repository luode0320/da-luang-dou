# 后台系统结构方案 v1

## 目标

本方案定义 `admin/` 后台系统应该具备的页面结构、业务能力和发布流程。后台不是简单管理页，而是本项目的内容生产中枢，负责把“素材、配置、规则、预览、校验、发布”串成完整闭环。

## 核心原则

1. 后台只负责内容编辑、校验、预览、发布和回滚。
2. 游戏侧只读取后台发布后的结果，不直接读取草稿。
3. 能配置的内容尽量不写死在代码里。
4. 后台以中文显示为主，存储层仍保留稳定 ID。
5. 后台必须支持目录选择、实体选择、素材绑定和版本管理。
6. 游戏首页中角色、怪物和 BOSS 的预览说明入口也必须由后台配置，展示内容、展示顺序、可见状态和绑定素材都不在游戏侧写死。

## 一、后台信息架构

### 1.1 顶层栏目

后台建议分为以下顶层栏目：

- 总览
- 首页配置
- 模块管理
- 全局资源库
- 素材资源库
- 配置编辑器
- 预览与校验
- 发布与回滚
- 调试与日志
- 权限与审计
- 模板中心
- 系统设置

### 1.2 主页总览

总览页应展示：

- 当前发布版本
- 待发布草稿数量
- 校验失败项数量
- 当前模块数量
- 当前角色、怪物、地图、技能、素材数量
- 最近发布记录
- 最近回滚记录
- 最近错误日志

### 1.3 首页配置

首页配置页面用于配置玩家进入游戏后可看到的轻量展示入口。

应支持：

- 开启或关闭首页预览入口
- 配置入口名称、入口图标和入口排序
- 选择展示哪些角色、怪物和 BOSS
- 配置展示顺序、推荐标记和分组标签
- 配置实体是否公开可见
- 配置是否按模块进度、关卡进度或遭遇记录解锁展示
- 绑定展示用图标、立绘、动作帧、技能图标和技能特效预览
- 配置简介、技能说明、所属模块来源和提示文案

## 二、核心业务页面

### 2.1 模块管理

模块管理页面用于管理每个游戏模块的完整配置。

应支持：

- 新建模块
- 复制模块
- 归档模块
- 编辑模块总配置
- 编辑模块节点
- 编辑模块难度
- 编辑模块奖励
- 绑定模块地图
- 绑定模块素材包
- 查看模块发布历史

### 2.2 全局资源库

全局资源库用于维护跨模块通用内容。

应支持：

- 角色库
- 怪物库
- BOSS 库
- 地图库
- 技能库
- 道具库
- 掉落表
- 刷怪表
- 规则包
- 文案包
- 无障碍设置包
- 音频包
- 首页展示包

### 2.3 素材资源库

素材资源库用于管理可被实体目录引用的素材。

应支持：

- 上传素材
- 浏览素材目录
- 按实体绑定素材
- 预览帧动画
- 预览图标
- 预览音频
- 查看素材来源与许可
- 查看素材引用关系
- 标记占位图与正式图

### 2.4 配置编辑器

配置编辑器应按实体类型拆分，而不是做成一个超级表单。

建议包含：

- 首页展示编辑器
- 角色编辑器
- 怪物编辑器
- BOSS 编辑器
- 地图编辑器
- 技能编辑器
- 道具编辑器
- 掉落编辑器
- 刷怪编辑器
- 关卡编辑器
- 难度编辑器
- 设置编辑器
- 音频编辑器
- 埋点编辑器
- 无障碍编辑器

## 三、关键编辑器能力

### 3.1 首页展示编辑器

支持编辑：

- 入口中文显示名
- 稳定 ID
- 首页入口图标
- 展示实体列表
- 展示实体类型
- 展示排序
- 分组标签
- 可见状态
- 解锁规则
- 展示用动作帧绑定
- 技能说明展示字段
- 所属模块来源

### 3.2 角色编辑器

支持编辑：

- 中文显示名
- 稳定 ID
- 固定武器
- 固定技能
- 动作帧绑定
- UI 图标
- 语音与音效
- 角色说明

### 3.3 怪物编辑器

支持编辑：

- 中文显示名
- 稳定 ID
- 类型
- 技能组
- 形态阶段
- 难度开放映射
- 掉落配置
- 动作帧绑定
- 特效绑定

### 3.4 地图编辑器

支持编辑：

- 中文显示名
- 稳定 ID
- 地图主题
- 瓦片资源
- 场景道具
- 区域划分
- 刷怪点
- 机关点
- 地图机制

### 3.5 技能编辑器

支持编辑：

- 中文显示名
- 稳定 ID
- 技能类型
- 冷却时间
- 施放条件
- 命中判定
- 特效资源
- 音效资源
- 动画资源

### 3.6 关卡编辑器

支持编辑：

- 模块归属
- 关卡编号
- 节点顺序
- 持续时间
- 刷怪规则
- Boss 出现条件
- 奖励规则
- 难度映射

## 四、后台编辑流

后台编辑流程建议统一为：

1. 创建草稿
2. 选择模块或全局目录
3. 选择实体类型
4. 选择实体目录
5. 选择主配置
6. 自动带出素材和依赖
7. 编辑覆盖项
8. 自动校验
9. 预览确认
10. 提交发布
11. 生成发布包

## 五、后台校验中心

校验中心负责把配置问题尽量挡在发布前。

应支持：

- 必填字段检查
- 类型检查
- 数值范围检查
- ID 冲突检查
- 引用缺失检查
- 素材缺失检查
- 循环引用检查
- 难度映射完整性检查
- 关卡节点断链检查
- 存档兼容风险检查

校验结果应分级：

- 阻断
- 警告
- 提示

## 六、后台预览中心

预览中心用于快速确认“配完后长什么样”。

应支持：

- 角色动作预览
- 怪物动作预览
- 技能特效预览
- 地图缩略预览
- 关卡节点预览
- 首页角色与怪物展示预览
- 掉落概率预览
- 音频试听
- 中文文案预览
- 设置项预览

## 七、发布与回滚

### 7.1 发布流程

推荐发布流程：

1. 校验通过
2. 预览确认
3. 生成发布包
4. 记录版本号
5. 发布到游戏侧可读目录
6. 写入发布日志

### 7.2 回滚能力

后台必须支持回滚到上一个稳定版本，至少保留：

- 当前版本
- 上一稳定版本
- 最近一次发布差异
- 回滚原因记录

## 八、权限与审计

后台应支持最少以下权限：

- 编辑
- 审核
- 发布
- 回滚
- 管理员

应记录以下审计信息：

- 谁改了什么
- 改了哪个实体
- 何时改的
- 改前改后差异
- 是否发布成功

## 九、模板中心

模板中心用于快速创建新模块和新实体。

应支持：

- 模块模板复制
- 角色模板复制
- 怪物模板复制
- 地图模板复制
- 技能模板复制
- 关卡模板复制

模板价值：

- 降低新模块创建成本
- 保持目录与字段一致
- 减少漏配
- 减少重复劳动

## 十、后台接口边界

后台后端建议至少提供以下能力：

- 配置读取
- 配置写入
- 首页展示配置读取
- 草稿保存
- 校验执行
- 预览生成
- 发布生成
- 回滚恢复
- 素材上传
- 素材检索
- 素材绑定
- 日志查询
- 审计查询

## 十一、前端页面建议

Vue 前端建议采用以下页面结构：

- `/dashboard`
- `/home-config`
- `/modules`
- `/resources`
- `/assets`
- `/editor/characters`
- `/editor/enemies`
- `/editor/maps`
- `/editor/skills`
- `/editor/items`
- `/editor/levels`
- `/editor/settings`
- `/preview`
- `/publish`
- `/rollback`
- `/logs`
- `/audit`
- `/templates`
- `/system`

## 十二、结论

后台的本质不是“管理配置”，而是：

`目录选择 -> 实体编辑 -> 素材绑定 -> 自动校验 -> 预览确认 -> 发布版本 -> 回滚追踪`

只要后台把这条链路做完整，前面提到的全配置驱动、中文命名、目录化组织和素材绑定就都能真正落地。
