# 阶段 3：Godot AI MCP 地图构建 + 后台配置游戏模块地图

## 阶段目标

阶段 3 负责模块地图。通过 Godot AI MCP 构建可玩的 2D 地图场景，并通过后台配置地图主题、地图素材、区域、碰撞、刷怪点、事件点和机关点。

地图不应只是单张背景图，而应拆成底图、props、碰撞、区域和可编辑层。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 地图场景 | Godot 中构建模块地图场景，支持角色移动、边界、碰撞和可视层。 |
| 地图层 | 支持底图、装饰层、交互层、碰撞层、区域层。 |
| 刷怪点 | 地图提供刷怪区域和刷怪点，供关卡配置引用。 |
| 事件点 | 地图可配置机关、天气、障碍、事件区域。 |
| 地图预览 | 后台和游戏侧可预览地图缩略图。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 地图基础信息 | 地图 ID、名称、主题、所属模块、尺寸。 |
| 地图素材 | 背景、瓦片、props、装饰、区域标记。 |
| 地图区域 | 出生点、可移动区域、禁行区域、刷怪区域、事件区域。 |
| 地图机制 | 机关、天气、光照、地形限制。 |
| 地图引用 | 模块和关卡可选择地图 ID。 |

## 配置目录建议

```text
game/data/modules/<module_id>/maps/<map_id>/
├── config.yaml                                # 地图主配置
├── assets/                                    # 地图总素材
├── tiles/                                     # 瓦片或地形资源
├── props/                                     # 场景道具
├── zones/                                     # 区域、碰撞、刷怪点定义
└── overrides/                                 # 模块局部覆盖
```

## 地图字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 地图稳定 ID。 |
| `name_zh` | 地图中文名。 |
| `theme` | 地图主题。 |
| `background_asset_id` | 背景素材。 |
| `tile_pack_id` | 瓦片资源包。 |
| `prop_pack_id` | 场景道具包。 |
| `zone_pack_id` | 区域定义。 |
| `spawn_pack_id` | 刷怪点定义。 |
| `event_pack_id` | 地图事件定义。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-3-001 | Godot 地图不是单张背景图，至少有底图、props、碰撞和区域。 |
| AC-3-002 | 后台能配置地图素材、区域、刷怪点和事件点。 |
| AC-3-003 | 关卡配置能引用地图 ID。 |
| AC-3-004 | 地图资源通过目录选择和素材绑定完成，不手写散乱路径。 |
| AC-3-005 | 地图可被 Godot AI MCP 构建和后续关卡系统消费。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 阶段 2 模块选择、模块 ID、模块主题。 |
| 输出给阶段 4 | 角色出生点、移动边界、角色动作测试地图。 |
| 输出给阶段 5 | 怪物刷怪区域、Boss 区域、怪物路径约束。 |
| 输出给阶段 6 | 关卡可引用地图、刷怪点和事件点。 |
