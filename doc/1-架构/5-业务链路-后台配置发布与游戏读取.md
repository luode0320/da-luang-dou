# 业务链路-后台配置发布与游戏读取

## 文档说明

- 业务目标: 让游戏内容经过后台编辑、校验、发布后，以标准化 YAML 配置包形式被游戏运行时安全读取。
- 入口: 后台发布入口；游戏启动、进入模块或进入关卡前的配置读取入口
- 责任模块: 后台前端、后台后端、发布包层、游戏运行层
- 事实来源: `项目设计.md`、`附录-全配置驱动完善方案.md`、`附录-后台系统结构方案.md`、`附录-游戏侧配置读取与合并规范.md`、`附录-版本矩阵与兼容判定规范.md`
- 最近同步时间: 2026-06-28 16:12:20
- 状态: 启用

## 前置条件

- 后台已存在待发布的配置草稿。
- 草稿中的实体、字段、引用关系和素材绑定已通过校验。
- 游戏运行层能够访问后台发布后的标准化配置结果。

## 主链路

1. 后台前端按实体类型编辑内容，并组织模块、全局资源、首页展示和素材绑定关系。
2. 后台后端在发布入口执行发布前校验，并在这一节点进入字段、引用、兼容和回滚可行性等相邻专题判断。
3. 校验通过后，后台后端把编辑态草稿整理为发布契约态结果，生成带版本号的标准化发布配置包。
4. 发布结果写入运行时唯一正式输入根目录，并同时沉淀运行时读取入口与发布审计入口。
5. 游戏启动、进入模块或进入关卡前，先确定目标发布版本，再进入正式版本判定与正文定位入口。
6. 游戏运行层在通过入口判定后消费目标模块所需正式正文配置，并把后续运行时消费动作转交相邻运行时专题。
7. 若存在自动存档恢复、发布后验证或回滚验证场景，运行时继续进入兼容判断、恢复判定和验证输出流程，再决定是否进入目标入口。
8. 运行时沉淀当前版本、目标场景、恢复结果和追踪输出，供测试、回滚和最终放行复核。

## 异常分支

| 异常场景 | 处理方式 | 最终状态 |
| --- | --- | --- |
| 后台校验失败 | 阻止发布并返回失败项 | 保持草稿状态 |
| 发布版本缺少正式版本入口、兼容入口或关键正文定位入口 | 视为无效发布包，不进入游戏运行输入 | 阻断读取 |
| 目标游戏版本低于发布包要求，或读取器不支持当前 `config_schema_version` | 视为版本矩阵不成立 | 阻断发布或阻断读取 |
| 发布包缺字段或引用失效 | 游戏侧明确报错并标记问题实体与来源目录 | 降级或阻断加载 |
| 新版本配置与旧存档不兼容 | 后台发布前声明迁移方案，游戏侧按兼容规则处理 | 进入兼容或迁移路径 |

## 结论

- 本文只维护“后台草稿如何进入发布契约态、游戏运行时在哪些节点进入正式入口与恢复验证、恢复验证在主链路中位于哪里”这条流程骨架。
