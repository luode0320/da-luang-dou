# doc/5-tests

测试验证资产目录。

## 当前入口

- 当前目录已建立，尚无当天时间戳测试目录；后续每一轮独立验证都从这里创建新的时间戳根目录。

## 目录现状

- 当前仓库尚未进入真实功能验证阶段，因此这里只保留目录规则和命名约束。
- 阶段 7 前重点需要补配置冒烟、关卡直达、存档兼容、回滚和 Windows 打包验证资产。

## 约定

- 验证资产默认放在当天时间戳目录下。
- 测试说明、回归记录、验证脚本和临时验证资产都归档到这里。
- 阶段 7 前重点补配置冒烟、关卡直达、存档兼容、回滚和 Windows 打包验证。
- 新增独立测试轮次时，按 `doc/5-tests/YYYY-MM-DD_HHmmss/<测试任务主题>/README.md` 结构落地。
- 默认新建当天时间戳目录，不把新一轮测试资产写入历史时间戳目录。
- 同一测试轮次的补充说明、脚本、截图和日志继续沿用原时间戳根目录，不另起并行轮次目录。
