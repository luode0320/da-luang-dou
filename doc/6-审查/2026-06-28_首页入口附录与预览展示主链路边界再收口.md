# 首页入口附录与预览展示主链路边界再收口

## 目的

- 继续收紧 `附录-首页预览展示包与首页入口规范.md` 与 `7-业务链路-首页预览配置与展示.md` 的正文边界。
- 让“首页控制层字段契约”和“首页预览展示主链路顺序”只各保留一份真相源。

## 本轮发现的重叠点

- 首页附录虽然已经主要负责字段和边界，但仍夹带少量“后台如何绑定、发布后如何进入首页展示”的流程语义。
- 7 号主链路本来已经负责“后台配置 -> 发布 -> 首页读取 -> 玩家进入预览”的实际顺序。
- 结果是首页附录和首页主链路都在回答“展示链路怎么走”。

## 本轮收口动作

### 首页附录

- 保留：
  - `home.yaml`
  - `shared_nav.yaml`
  - `home_preview/<preview_pack_id>/config.yaml`
  - 剧透等级
  - 首页与实体正文边界
  - 发布前最小校验集合
- 删除或下沉：
  - 后台如何绑定首页展示包
  - 发布后如何进入首页展示
  - 玩家进入预览后的逐步加载顺序

### 首页预览主链路

- 保持它继续作为：
  - 后台绑定
  - 发布写入控制层
  - 首页读取
  - 玩家进入预览后按顺序展示
  的唯一正文
- 仅同步了一处边界表述，让它更明确承接“字段契约之后的流程顺序”。

## 当前边界

### 首页预览展示包与首页入口附录

- 唯一负责：首页入口字段、公共导航字段、预览展示包字段、剧透等级、实体引用边界和展示校验。

### 首页预览配置与展示主链路

- 唯一负责：首页预览内容如何从后台配置进入发布结果，再进入首页展示和预览展示顺序。

### 首页设置主链路

- 继续负责：首页设置入口进入后的默认设置、个人偏好合并和运行时操作链路，不与首页预览展示主链路混写。

## 结果

- 首页附录不再承担展示步骤顺序正文。
- 首页预览主链路继续成为首页展示流程的唯一正文真相源。
- 首页附录、首页预览主链路、首页设置主链路三者边界进一步稳定。
