# 首页预览展示包与首页入口规范

## 目标

本附录用于把阶段 1 中已经提出、但尚未在长期架构层写硬的“首页入口、首页预览展示包、公共导航和剧透边界”统一成长期规则。

它重点回答以下问题：

- 首页入口配置应该承载哪些最小字段
- `home.yaml`、`shared_nav.yaml` 和 `home_preview/<preview_pack_id>/config.yaml` 如何分层
- 首页预览展示包与角色、怪物、Boss 正文配置如何解耦
- 首页预览为什么只是轻量展示入口，而不是完整图鉴
- 首页入口、首页预览和设置入口如何进入统一控制配置契约

## 一、分层总原则

### 1. 首页入口配置是控制层入口，不复制实体正文，也不代写展示主链路

- 首页控制配置负责“首页显示什么入口、入口顺序是什么、哪些入口可见、预览展示包引用哪个配置”。
- 角色、怪物、Boss、技能说明和动画资源的正文仍归实体正文配置承载。
- 不允许把角色、怪物、Boss 的详细正文字段平铺复制进首页入口配置。
- “后台如何绑定、发布后如何进入首页展示、玩家进入预览后如何按顺序展示”继续转交 [7-业务链路-首页预览配置与展示](7-业务链路-首页预览配置与展示.md)。

### 2. 首页预览展示包只负责“展示谁、按什么规则展示”，不负责“实体是什么”

- 首页预览展示包长期只负责展示顺序、分组、剧透等级、过滤条件和实体引用。
- 实体本体属性、技能正文、固定武器、Boss 形态和解锁规则由各自实体正文维护。
- 首页预览展示包引用实体 ID，不复制实体正文，不单独维护第二套实体定义。
- 首页预览展示包不负责“玩家点进入口后先显示哪一屏、按什么主链路逐步加载”的流程顺序。

### 3. 首页预览不是完整图鉴

- 首页预览长期定位为轻量展示入口。
- 它可以承接角色、怪物、Boss、技能说明和动作预览。
- 它不承接完整图鉴、收藏率、隐藏条目、成就联动、百科系统或长期收集统计。

## 二、控制配置载体分层

### 2.1 `home.yaml`

`home.yaml` 负责首页主入口配置，长期至少回答以下问题：

- 首页标题和主视觉引用是什么
- 首页有哪些入口按钮
- 入口顺序、文案、图标和显示状态是什么
- 首页是否展示模块入口、预览入口和设置入口
- 首页主入口不负责解释后续展示流程顺序，只负责入口和引用关系。

### 2.2 `shared_nav.yaml`

`shared_nav.yaml` 负责公共导航和跨页面共享入口，长期至少回答以下问题：

- 首页、模块选择、设置、帮助等公共入口的导航标识
- 哪些导航项允许在首页显示
- 公共导航的顺序、分组和复用策略

### 2.3 `home_preview/<preview_pack_id>/config.yaml`

首页预览展示包负责“首页预览入口点进去之后看什么”，长期至少回答以下问题：

- 这个展示包的稳定 ID 是什么
- 展示哪些实体
- 按什么顺序和分组展示
- 展示哪些技能说明和动作预览
- 当前展示包的剧透等级和可见性约束是什么

它不负责：

- 发布时按什么步骤写入控制层
- 运行时按什么步骤读取并展示
- 玩家进入预览后的逐步加载顺序

## 三、首页主入口最小字段契约

建议 `game/data/control/home.yaml` 至少包含：

| 字段 | 作用 |
| --- | --- |
| `home_id` | 首页配置稳定 ID |
| `title_text` | 首页主标题 |
| `subtitle_text` | 首页副标题或提示语 |
| `logo_asset_id` | 首页 Logo 资源引用 |
| `background_asset_id` | 首页背景资源引用 |
| `entry_items` | 首页入口按钮集合 |
| `default_focus_entry_id` | 默认聚焦入口 |
| `preview_pack_id` | 首页预览展示包引用 |
| `settings_entry_visible` | 首页设置入口是否显示 |
| `module_entry_visible` | 首页模块入口是否显示 |

### `entry_items` 最小字段建议

| 字段 | 作用 |
| --- | --- |
| `entry_id` | 首页入口稳定 ID |
| `entry_type` | 入口类型，如 `continue`、`new_game`、`module_select`、`preview`、`settings` |
| `entry_title` | 入口标题 |
| `entry_icon_asset_id` | 入口图标引用 |
| `display_order` | 展示顺序 |
| `visible` | 是否显示 |
| `enabled_rule_id` | 启用条件引用 |
| `target_ref` | 入口跳转目标引用 |

## 四、公共导航最小字段契约

建议 `game/data/control/shared_nav.yaml` 至少包含：

| 字段 | 作用 |
| --- | --- |
| `nav_version` | 公共导航版本 |
| `nav_items` | 导航项集合 |
| `default_home_entry_id` | 默认首页导航入口 |
| `settings_route_ref` | 设置页路由引用 |
| `module_route_ref` | 模块选择页路由引用 |

### `nav_items` 最小字段建议

| 字段 | 作用 |
| --- | --- |
| `nav_id` | 导航稳定 ID |
| `nav_title` | 导航文案 |
| `nav_icon_asset_id` | 导航图标 |
| `route_ref` | 路由或页面目标引用 |
| `display_order` | 顺序 |
| `visible` | 是否显示 |
| `entry_scope` | 生效页面范围 |

## 五、首页预览展示包最小字段契约

建议 `game/data/control/home_preview/<preview_pack_id>/config.yaml` 至少包含：

| 字段 | 作用 |
| --- | --- |
| `preview_pack_id` | 展示包稳定 ID |
| `title_text` | 展示包标题 |
| `summary_text` | 展示包说明 |
| `visible_entity_refs` | 展示实体 ID 列表 |
| `entity_type_filter` | 展示实体类型过滤 |
| `group_configs` | 展示分组配置 |
| `display_order` | 展示包在首页入口中的顺序 |
| `spoiler_level` | 剧透等级 |
| `unlock_rule_id` | 展示包解锁规则 |
| `preview_asset_refs` | 首页预览专用素材引用 |
| `status` | `active` / `disabled` / `draft` |

### `visible_entity_refs` 约束

- 只允许引用已存在的角色、怪物、Boss 或其他允许展示的实体 ID。
- 只负责引用，不复制实体正文。
- 被引用实体若 `preview_visible=false` 或状态不可展示，发布时应校验阻断或自动剔除异常项。

### `group_configs` 最小字段建议

| 字段 | 作用 |
| --- | --- |
| `group_id` | 分组稳定 ID |
| `group_title` | 分组标题 |
| `group_type` | 分组类型，如角色、怪物、Boss |
| `entity_refs` | 当前分组引用的实体 ID |
| `display_order` | 分组顺序 |
| `collapsed_by_default` | 默认是否折叠 |

## 六、剧透等级与可见性规则

长期建议至少支持以下剧透等级：

- `public`
- `module_hint`
- `boss_spoiler`
- `hidden`

含义：

- `public`：允许默认展示。
- `module_hint`：允许展示与模块主题相关的预告信息。
- `boss_spoiler`：涉及 Boss 信息，默认应受额外显示控制。
- `hidden`：仅调试、验证或后台预览可见。

统一约束：

- 剧透等级只决定是否允许展示，不改写实体正文。
- `hidden` 项不得在正式玩家首页默认展示。
- 首页预览中展示 Boss 时，必须显式经过剧透等级与可见性控制。

## 七、首页预览与实体正文的边界

### 7.1 角色

- 首页预览可引用角色头像、立绘、动作预览、说明和固定技能摘要。
- 角色固定武器、固定技能、动画包和解锁规则仍以角色正文为准。

### 7.2 怪物与 Boss

- 首页预览可引用普通怪、精英怪、Boss 的预览卡片、动作预览和说明。
- 怪物技能集、Boss 终极形态、狂暴概率和掉落绑定仍以怪物 / Boss 正文为准。

### 7.3 技能说明

- 首页预览中的技能说明只做展示摘要。
- 技能真实行为、数值、冷却和动画事件位点仍由技能正文或实体绑定配置承载。

## 八、首页入口与其他长期专题的边界

### 8.0 首页控制层边界速查

| 载体 | 本文负责 | 应转交的专题 |
| --- | --- | --- |
| `home.yaml` | 首页标题、首页入口集合、入口顺序、首页预览展示包引用、设置入口可见性。 | 模块是否可解锁与如何继续上次，转交 [附录-模块索引与启动入口规范](附录-模块索引与启动入口规范.md)。 |
| `shared_nav.yaml` | 公共导航、跨页面共享入口、首页可见导航项。 | 模块正文、模块名单和模块内默认地图，转交 [附录-模块正文配置与模块名单规范](附录-模块正文配置与模块名单规范.md)。 |
| `home_preview/<preview_pack_id>/config.yaml` | 首页预览展示包、剧透等级、实体预览分组与展示摘要。 | 实体正文、角色固定武器、怪物技能层级和 Boss 形态，分别转交角色 / 怪物 / 技能相关附录。 |

### 8.1 与相邻专题的最小边界

- 首页只负责“首页显示哪些入口、展示哪个预览包、设置入口是否可见”。
- 模块是否可进入、继续上次与开始新游戏，统一转交 [附录-模块索引与启动入口规范](附录-模块索引与启动入口规范.md)。
- 默认设置、玩家个人设置、暂停和加速，统一转交 [附录-玩家设置与运行时操作规范](附录-玩家设置与运行时操作规范.md)。
- 后台如何绑定首页展示包、发布后如何进入首页、玩家进入预览后如何按顺序展示，统一转交 [7-业务链路-首页预览配置与展示](7-业务链路-首页预览配置与展示.md)。
- 若后续新增完整图鉴，应单独建长期专题，不复用首页预览展示包承载收集系统。

### 8.4 本文不负责的事项

- 不负责 `modules_index.yaml` 的模块解锁规则、难度入口和继续上次决策。
- 不负责 `module.yaml`、`role_roster.yaml`、`enemy_roster.yaml`、`map_roster.yaml` 的模块正文与名单边界。
- 不负责角色固定技能、固定武器、Boss 阶段技能和状态效果正文。

## 九、发布前最小校验集合

| 校验项 | 通过条件 | 阻断条件 |
| --- | --- | --- |
| 首页主入口完整性 | 首页标题、入口集合、目标引用和展示包引用齐全 | 关键字段缺失 |
| 公共导航合法性 | 导航项可解析，顺序与路由引用完整 | 导航断链 |
| 展示包引用合法性 | `preview_pack_id` 能解析到合法展示包 | 展示包不存在 |
| 实体引用合法性 | `visible_entity_refs` 均能解析到合法实体 | 实体引用断链 |
| 剧透边界合法性 | Boss 和隐藏内容遵守 `spoiler_level` 规则 | 隐藏内容误公开 |
| 预览资源可解析 | 预览图、动作预览和图标引用存在 | 素材断链 |
| 轻量边界成立 | 未引入图鉴收集率、成就联动、隐藏条目统计 | 范围膨胀为完整图鉴 |

## 十一、结论

- 首页附录只保留首页主入口、公共导航和首页预览展示包三层契约。
- 进入模块、进入设置和进入图鉴之外系统，统一通过引用其他专题承接，不在本文重复扩写。
- 展示链路顺序继续只留在首页预览主链路正文，不在本文平行维护。
