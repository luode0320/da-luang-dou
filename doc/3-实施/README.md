## 落点规则

- 实施总览、实施周期、阶段推进和阶段实施拆分统一收口到 `doc/3-实施/`。
- 当前重点仍是阶段 0 配置驱动基础设施，后续阶段文档以阶段 0 的配置、校验、发布和运行时读取能力为前置依赖。
- 若文档按阶段编号组织，并且同时覆盖游戏侧目标与后台配置目标，优先放入 `doc/3-实施/`。
- 若内容是单一需求的编码前总体实施方案，也优先放入 `doc/3-实施/`，不要继续混写回 `doc/2-需求/`。
- 与单一需求绑定的实施周期文档，默认沿用 `<需求文档主干>实施周期NN_中文主题.md` 形式继续维护。
- 当历史实施总览已经分散且用户明确要求重新整理时，允许新建 dated 正式实施总览收口当前有效口径，并把旧实施总览保留为历史来源。
- 实施目录导航默认先区分“阶段导航主入口”“按需求归类的实施总览”“按阶段归类的实施文档”，避免不同职责的实施文档继续平铺混列。
