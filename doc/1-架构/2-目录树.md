# 目录树

## 文档说明

- 用途: 记录当前仓库中长期有效的目录结构、目标发布包结构和路径级职责边界。
- 事实来源: 当前仓库真实目录、`README.md`、`AGENTS.md`、`项目设计.md`
- 适用范围: 仓库根目录及 `game/`、`admin/`、`tools/windows/`、`doc/`
- 最近同步时间: 2026-06-28 20:05:00

## 本文负责什么

- 说明当前真实存在的目录骨架。
- 说明后续发布包建议采用的路径结构。
- 说明每条路径允许放什么、不应该放什么。
- 作为路径级真相源，回答“哪个目录承接哪类内容”。

## 本文不负责什么

- 不展开模块之间的输入输出和依赖方向，那属于 [模块职责](3-模块职责.md)。
- 不展开某条业务链路如何流转，那属于 [主要业务链路](4-主要业务链路.md) 与对应 `5-9` 链路文档。
- 不展开 `module.yaml`、技能正文、武器正文、状态效果正文等专题字段契约，那属于对应附录。

## 目标发布包结构（建议）

以下结构用于指导后续阶段 0 和运行时实现，属于目标契约设计，不是当前磁盘已经完整存在的事实：

```text
game/data/published/
└── <release_version>/
    ├── manifest.yaml
    ├── compatibility.yaml
    ├── indexes/
    │   ├── modules.yaml
    │   ├── entities.yaml
    │   └── assets.yaml
    ├── control/
    ├── global/
    └── modules/
```

## 路径级边界

- `game/data/control/` 只回答首页、公共导航、模块入口、调试入口等控制层问题，不承接角色、怪物、技能、武器、道具等实体正文。
- `game/data/global/` 只承接跨模块公共默认值、公共实体和公共规则，不承接某个模块专属覆盖。
- `game/data/modules/` 只承接单模块正文、单模块名单、单模块关卡、单模块地图和单模块覆盖，不承接跨模块公共默认值。
- `game/data/published/` 只承接后台发布后的版本化只读配置包，不承接后台草稿和编辑态目录。
- `doc/1-架构/` 到 `doc/7-验收/` 只按研发文档职责分层，不与 `game/`、`admin/`、`tools/windows/` 混放生产资产。

## 目录职责

| 路径 | 职责 | 允许内容 | 禁止内容 |
| --- | --- | --- | --- |
| `admin/backend/` | 后台配置系统 Go 后端 | 配置服务、校验、发布、回滚、审计接口 | 游戏运行逻辑 |
| `admin/frontend/` | 后台配置系统 Vue 前端 | 配置编辑器、预览、发布页面、后台工作台 | Godot 场景或运行态脚本 |
| `game/` | Godot 4 主游戏工程 | 场景、脚本、资源、运行态逻辑 | 后台内容编辑逻辑 |
| `game/data/control/` | 总层控制命名空间骨架 | 首页入口、公共导航、首页预览展示包、模块入口和调试入口等控制层配置 | 把后台草稿结构或实体正文直接塞进控制层 |
| `game/data/global/` | 全局公共配置命名空间骨架 | 跨模块公共默认值、公共实体、公共规则和公共资源引用 | 模块专属覆盖内容 |
| `game/data/modules/` | 单模块配置命名空间骨架 | 单模块正文、单模块名单、单模块地图、单模块关卡、单模块奖励与模块级覆盖 | 跨模块公共默认值 |
| `game/data/published/` | 后台发布产物根入口 | 版本化标准化只读配置包、索引、兼容信息 | 草稿或后台编辑态结构 |
| `doc/1-架构/` | 长期架构专题 | 目录树、模块职责、主链路、架构附录 | 当前实施步骤、测试证据 |
| `doc/2-需求/` | 需求域文档 | 主需求、需求索引、配套 SVG、需求图片资产 | 实施总览、最终验收 |
| `doc/3-实施/` | 实施域文档 | 实施总览、实施周期、阶段拆分和推进 | 长期架构专题 |
| `doc/4-bugs/` | Bug 域文档 | 单问题目录、复现、根因、验证 | 测试轮次总目录 |
| `doc/5-tests/` | 测试域文档 | 时间戳测试轮次、验证脚本、回归记录 | Bug 主记录 |
| `doc/6-审查/` | 审查域文档 | 结构归位、规则核对、实现审查结论 | 需求主入口 |
| `doc/7-验收/` | 验收域文档 | 验收标准、最终验收、放行结论 | 实施过程记录 |
| `tools/windows/` | Windows 发布支持 | 打包、安装器、渠道参数、分发与安装验证辅助脚本 | 游戏主逻辑或后台业务逻辑 |

具体载体文件名、字段定义和覆盖优先级，继续转交相邻附录维护；本文只保留“哪类内容应该落在哪条路径”的路径级真相源。

## 变更同步

- 新增顶层目录时，同步更新 [总架构](1-总架构.md) 与本文件。
- `game/data/` 的层级变化会同时影响 [模块职责](3-模块职责.md) 与相关业务链路文档。
- 路径级语义变化时，只在本文件记录“哪个目录承接什么”；具体载体、字段和覆盖规则继续同步到相邻附录。
- 发布包目录骨架变化时，同步更新发布契约相关附录与 [业务链路-后台配置发布与游戏读取](5-业务链路-后台配置发布与游戏读取.md)。
- Windows 打包、安装器或渠道路径变化时，同步更新 [附录-Windows打包安装与分发规范](附录-Windows打包安装与分发规范.md)。
- 文档目录规则变化时，优先同步 `doc/README.md` 和对应子目录 `README.md`，再回写长期记忆。
- 若旧截图、历史讨论或旧链接仍引用 `2026-06-22_*` 架构文档名称，优先把引用改到对应 `附录-*` 文档，不保留并行旧入口。
