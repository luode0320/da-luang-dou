# doc/4-bugs

Bug 记录目录。

## 当前入口

- 当前目录已建立，尚无实际 Bug 根目录；后续每个独立问题都从这里开始归档。

## 目录现状

- 当前仓库仍处于规则、设计文档和目录骨架阶段，尚未形成真实 Bug 归档记录。
- 该目录继续保持有效占位，不补造假问题文档。

## 约定

- 每个独立 Bug 使用一个根目录，命名格式为 `YYYY-MM-DD_HHmmss_问题中文简介/`。
- Bug 根目录内默认包含 `README.md`、`flow.svg`、`sequence.svg`。
- 同一个 Bug 从录入、复现、根因、验证到收口默认沿用同一个根目录。
- Bug 主记录不混写到 `doc/5-tests/` 或根目录临时文档中；测试证据按需要链接到对应测试目录即可。
- 后续若出现真实 Bug，先创建问题根目录，再补 `README.md`、`flow.svg`、`sequence.svg` 三件套，不单独在根目录平铺说明文件。
