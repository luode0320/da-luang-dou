# 阶段 1：主页首页设计 + 后台配置主页首页管理

## 阶段目标

阶段 1 负责游戏首页。它不是复杂大厅，也不是完整图鉴系统，而是玩家进入游戏后看到的第一层入口：继续上次、开始新游戏、模块入口、角色怪物预览、设置入口。

后台侧必须能配置首页展示内容，游戏侧只读取后台发布后的首页配置。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 首页主界面 | 展示游戏标题、继续上次、开始新游戏、模块入口、角色怪物预览入口、设置入口。 |
| 首页预览入口 | 玩家可以查看后台选择的角色、怪物、Boss、技能说明和动作预览。 |
| 继续上次 | 读取当前模块自动存档，进入模块选择或模块节点。 |
| 开始新游戏 | 重置当前模块自动存档，从模块起点开始。 |
| 设置入口 | 至少预留音量、分辨率、语言、键位入口，第一版可只做基础展示。 |
| 范围边界 | 不做完整图鉴、成就、复杂商店、复杂首页、多模式入口。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 首页入口配置 | 配置首页按钮、入口顺序、显示状态、入口图标和入口文案。 |
| 首页预览配置 | 选择展示哪些角色、怪物、Boss，配置排序、分组、技能字段和剧透等级。 |
| 首页文案配置 | 配置标题、按钮、提示、空状态、错误提示和确认弹窗文案。 |
| 首页素材绑定 | 配置背景图、Logo、按钮图标、角色怪物预览图和动作帧。 |
| 可见性控制 | 控制未完成、未发布、剧透型 Boss 是否展示。 |

## 配置目录建议

```text
game/data/control/
├── home.yaml                                  # 首页主入口配置
├── shared_nav.yaml                            # 公共导航配置
└── home_preview/<preview_pack_id>/
    ├── config.yaml                            # 首页角色怪物预览展示包
    └── assets/                                # 首页展示素材
```

## 后台字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 首页配置稳定 ID。 |
| `entry_title` | 首页入口标题。 |
| `entry_icon_asset_id` | 首页入口图标。 |
| `display_order` | 首页按钮排序。 |
| `visible_entity_refs` | 首页预览展示实体 ID 列表。 |
| `entity_type_filter` | 展示角色、怪物、Boss 的筛选类型。 |
| `unlock_rule_id` | 后续按进度解锁展示的规则。 |
| `spoiler_level` | 剧透等级。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-1-001 | 首页入口由 YAML 配置驱动，不在 Godot 代码中写死按钮内容。 |
| AC-1-002 | 首页可进入角色怪物预览入口。 |
| AC-1-003 | 首页预览内容来自后台配置的实体 ID。 |
| AC-1-004 | 首页预览不是完整图鉴，不显示收集率、成就或隐藏条目系统。 |
| AC-1-005 | 后台能配置首页入口显示、顺序、文案、素材和可见状态。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 配置基础设施、稳定 ID、UTF-8 文案、素材引用规范。 |
| 输出给阶段 2 | 首页进入模块选择的入口和模块索引读取能力。 |
