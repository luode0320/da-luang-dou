# 架构附录边界矩阵补强与重叠口径收紧

## 背景

- 当前 `doc/1-架构/` 已经形成 27 份附录，但部分专题的边界仍容易靠读者自行推断。
- 最容易重叠的两组专题是：
  - 首页入口 / 模块入口 / 模块正文
  - 技能 / 武器投射物 / 状态效果
- 如果只靠段落描述，后续很容易再次在新文档里把这些职责混写。

## 本轮处理结论

### 新增的边界强化方式

- 在 [doc/1-架构/README.md](../1-架构/README.md) 新增“专题边界速查”
- 在以下附录内新增“分层边界速查”或“本文不负责的事项”
  - `附录-首页预览展示包与首页入口规范.md`
  - `附录-模块索引与启动入口规范.md`
  - `附录-模块正文配置与模块名单规范.md`
  - `附录-技能配置与战斗表现规范.md`
  - `附录-武器投射物与命中效果规范.md`
  - `附录-状态效果与属性修饰规范.md`

### 本轮收紧后的两组边界

#### 控制配置层三件套

- 首页附录只管首页主入口、公共导航、首页预览展示包和剧透边界
- 模块索引附录只管模块入口显示、继续上次、开始新游戏、难度入口和解锁规则
- 模块正文附录只管 `module.yaml` 与 `role_roster.yaml` / `enemy_roster.yaml` / `map_roster.yaml`

#### 战斗载体三件套

- 技能附录只管技能实体本体、施法/飞行/命中/冷却语义和摘要展示
- 武器附录只管固定武器正文、投射物包、命中瞬时反馈和 `effect_refs` 引用入口
- 状态效果附录只管 `effects/<effect_id>/`、载荷、生命周期、叠层、驱散和清理

## 影响

- 这轮没有新增新的长期架构专题，而是优先强化已有附录的边界可读性。
- 后续继续优化架构设计时，可以优先围绕这些速查矩阵做去重和收口，而不是再新建平行专题。
