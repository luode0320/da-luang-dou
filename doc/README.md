# doc 文档目录

## 目录定位

`doc/` 是仓库级研发文档归档区，统一按目录职责管理架构、需求、实施、Bug、测试、审查和验收文档，避免不同阶段产物混放。

根入口分工如下：

- 根目录 [项目设计.md](../项目设计.md)：项目级最高入口，负责产品方向、总体设计和核心约束。
- [PROJECT_MEMORY.md](../PROJECT_MEMORY.md)：长期项目事实记忆入口，沉淀稳定事实。
- [PROJECT_STYLE.md](../PROJECT_STYLE.md)：长期项目风格记忆入口，沉淀命名、落点和写法样例。
- [doc/README.md](README.md)：`doc/` 导航入口，负责说明分类、当前主入口和维护规则。
- 各子目录 `README.md`：负责本目录的落点规则、当前入口和后续维护约定。

## 维护规则

- 新增主需求、需求索引、需求变更和需求配套 SVG 时放入 `doc/2-需求/`。
- 新增配置、后台、字段、读取、编码和跨阶段技术约束时放入 `doc/1-架构/`。
- 新增实施总览、实施周期、阶段拆分、阶段推进和阶段实施文档时放入 `doc/3-实施/`。
- 新增 Bug 记录、流程图、时序图和验证结论时放入 `doc/4-bugs/`。
- 新增测试说明、验证脚本、回归记录和临时验证资产时放入 `doc/5-tests/`。
- 新增结构归位、实现审查、格式清理、语法检查和注释核对结论时放入 `doc/6-审查/`。
- 新增验收标准、最终验收和放行结论时放入 `doc/7-验收/`。
- 新增文档前，先更新对应子目录 `README.md` 的“当前入口”或“目录说明”，保持导航可用。
