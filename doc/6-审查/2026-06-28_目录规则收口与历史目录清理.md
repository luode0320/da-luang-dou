# 目录规则收口与历史目录清理

## 背景

- 当前仓库已按共享 `artifact-storage-rules` 重新收口为 `doc/2-需求/`、`doc/1-架构/`、`doc/3-实施/`、`doc/7-验收/`、`doc/4-bugs/`、`doc/5-tests/`、`doc/6-审查/`。
- 历史上曾出现 `ment/` 目录和一段时间的 `doc/阶段/` 落点，但当前仓库已迁回共享 skill 口径。
- 若不把本轮迁移结果同步到项目入口文档，后续很容易再次按旧目录口径漂移。

## 当前结论

| 项目 | 结论 |
| --- | --- |
| 阶段与实施目录 | 本仓库统一使用 `doc/3-实施/`。 |
| 需求相关计划与验收 | 实施总览迁入 `doc/3-实施/`，验收文档迁入 `doc/7-验收/`，不再跟随 `doc/2-需求/` 混放。 |
| 历史 `ment/` | 视为废弃目录，不再作为正式入口，也不保留兼容层。 |
| 审查类归档 | 统一收口到 `doc/6-审查/`，不再新增根级 `review/`、`analysis/`。 |

## 本轮检查结果

- 工作区中已不存在真实 `ment/` 目录。
- 根 `README.md`、`项目设计.md`、`PROJECT_MEMORY.md`、`PROJECT_STYLE.md`、`doc/README.md` 已同步到当前目录结构。
- `AGENTS.md`、`README.md`、`项目设计.md`、`PROJECT_MEMORY.md`、`PROJECT_STYLE.md` 正在同步到共享 skill 目录口径。

## 后续规则

- 新增阶段文档、实施总览和实施周期时，直接进入 `doc/3-实施/`。
- 新增验收标准和最终验收时，直接进入 `doc/7-验收/`。
- 若再发现历史 `ment/` 文件或旧路径引用，默认执行“迁回 `doc/` 正式目录 + 删除旧入口”，不保留并行旧目录。
