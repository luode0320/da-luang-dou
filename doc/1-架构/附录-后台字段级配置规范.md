# 后台字段级配置规范 v1

## 目标

本规范用于定义后台中各类实体应该有哪些字段、哪些字段必须有、哪些字段可覆盖、哪些字段必须关联素材。它的目标是让后台编辑器不只是“能填表”，而是“能形成稳定、可发布、可追踪的配置结果”。

## 1. 通用字段模板

所有实体建议默认包含以下通用字段：

- `id`：稳定 ID
- `name_zh`：中文显示名
- `name_en`：英文预留名，可选
- `description`：说明
- `status`：草稿、已发布、归档
- `version`：版本号
- `module_id`：模块归属，若为全局实体则可为空
- `tags`：标签
- `assets`：素材引用列表
- `overrides`：覆盖字段
- `metadata`：扩展元数据

## 2. 角色字段

角色实体建议包含：

- `id`
- `name_zh`
- `description`
- `fixed_weapon_id`
- `fixed_skill_ids`
- `portrait_asset_id`
- `animation_pack_id`
- `voice_pack_id`
- `ui_icon_asset_id`
- `unlock_rule_id`

可覆盖项：

- 武器绑定
- 技能绑定
- 图标
- 动画包
- 语音包
- 解锁条件

## 3. 怪物字段

- `phase_ids`
- `loot_table_id`
- `animation_pack_id`
- `vfx_pack_id`
- `sfx_pack_id`

- 特效包

## 14. 字段级配置原则

- 必填字段必须能在后台界面上显式看到。
- 可覆盖字段必须标记“是否允许继承”。
- 素材关联字段必须支持选择器。
- 一切可追踪变更都要保留版本号。
- 任何字段更名都必须保留别名映射。

## 15. 结论

字段级配置的核心不是“字段越多越好”，而是：

- 让后台知道该填什么
- 让游戏侧知道该读什么
- 让版本兼容知道该怎么迁移
- 让素材绑定知道该关联什么
