# 全配置驱动完善方案

## 目标

本方案用于补齐“全配置驱动”落地时最容易遗漏的部分。目标不是只回答“内容放哪儿”，而是把以下问题一起收口：

- 配置如何命名与引用
- 配置如何继承与覆盖
- 配置如何校验
- 配置如何发布与回滚
- 配置如何绑定素材
- 配置如何预览与调试
- 配置如何兼容旧存档与旧版本

## 总原则

1. 所有可配置内容都应优先配置化，不把业务逻辑写死在代码里。
2. 需要素材绑定、多个实例、局部覆盖、模块复用、后台树形选择的内容，优先目录化。
3. 游戏侧只读取后台发布后的结果，不直接依赖后台草稿结构。
4. 代码只保留通用加载、校验、合并、运行、预览和渲染能力。
5. 任何配置化方案都必须同时考虑命名、引用、校验、发布、回滚、兼容和调试。

## 1. 配置引用规范

### 1.1 统一 ID

每个实体必须拥有稳定 ID，例如：

- `role_knight`
- `enemy_slime`
- `boss_ogre`
- `map_graveyard`
- `skill_fireball`
- `item_gold_sword`

### 1.2 引用方式

后台和游戏侧都只引用稳定 ID，不引用临时显示名。

推荐引用链路：

- 模块引用全局实体 ID
- 模块内实体可覆盖全局实体，但仍保留同类稳定 ID 规则
- 关卡、技能、掉落、刷怪表、素材绑定都通过 ID 关联
- 首页预览入口通过 ID 引用已发布或准备展示的角色、怪物、BOSS 和技能说明，不复制实体配置

### 1.3 命名约定

- ID 使用小写加下划线
- 同类实体 ID 不重复
- 显示名可以重复，但不能作为唯一标识
- 目录名尽量与 ID 对齐，避免后台选目录和运行时 ID 对不上

## 2. 配置继承与覆盖

### 2.1 合并优先级

最终生效顺序统一为：

`实体覆盖 > 模块覆盖 > 全局默认 > 内置默认值`

其中：

- `实体覆盖`：某个角色、怪物、地图、技能自己的局部覆盖
- `模块覆盖`：某个模块对全局实体或模块公共规则的覆盖
- `全局默认`：跨模块通用默认配置
- `内置默认值`：代码里的最后兜底值

### 2.2 字段覆盖规则

- 标量字段直接覆盖
- 字典字段按 key 合并
- 列表字段默认整表替换，除非显式声明支持追加或按 ID 合并
- 素材引用字段以最终发布结果为准

### 2.3 覆盖边界

不建议把所有字段都做成可继承，建议只对以下字段开放覆盖：

- 数值参数
- 技能启用与禁用
- 怪物阶段解锁
- 掉落概率
- 出怪表
- 模块规则
- 素材引用

不建议频繁覆盖的字段：

- 实体 ID
- 实体类型
- 目录归属
- 核心逻辑分支

## 3. 配置校验体系

### 3.1 基础校验

后台在保存和发布前必须校验：

- 必填字段是否齐全
- 字段类型是否正确
- 数值范围是否合法
- ID 是否重复
- 引用是否存在
- 素材路径是否可访问
- 是否存在循环引用

### 3.2 依赖校验

必须能检查以下依赖链：

- 角色是否绑定了合法武器与技能
- 技能是否绑定了合法特效与音效
- 怪物是否绑定了合法技能与素材
- 地图是否绑定了合法地形、区域和刷怪点
- 关卡是否能找到模块、地图、怪物池和奖励配置

### 3.3 发布前拦截

以下情况必须阻止发布：

- 缺少关键配置字段
- 引用不存在
- 素材缺失
- ID 冲突
- 难度映射不完整
- 关卡节点断链
- 旧存档无法识别的破坏性变更未声明迁移方案

## 4. 素材绑定与预览

### 4.1 绑定原则

- 素材跟着实体目录走
- 素材不通过手写路径作为主要工作方式
- 后台以“目录选择 + 素材列表 + 预览图”完成绑定

### 4.2 预览内容

后台至少应支持以下预览：

- 角色站姿和动作帧
- 怪物待机、移动、攻击、死亡动作帧
- 技能施放、命中、飞行、冷却效果
- 地图底图、瓦片、道具、区域
- UI 图标、掉落图标、技能图标
- 首页角色、怪物、BOSS 和技能说明展示效果

### 4.3 绑定结果

发布后应输出：

- 实体 ID
- 实体配置
- 素材引用表
- 依赖关系表
- 预览版本号

## 5. 发布与版本

### 5.1 草稿、已发布、归档

后台内容建议至少分三态：

- 草稿：可编辑
- 已发布：游戏侧读取
- 归档：历史保留，不再参与默认发布

### 5.2 发布包

每次发布应生成完整发布包，包含：

- 合并后的 YAML
- 资源引用清单
- 版本号
- 变更摘要
- 兼容说明

### 5.3 回滚

必须支持按版本回滚到上一次可用发布包，至少要保留：

- 最近一次稳定版本
- 最近一次发布记录
- 回滚差异说明

## 6. 存档兼容

### 6.1 原则

后续版本追加模块、角色、技能、怪物、道具或配置时，旧存档应尽量继续可用。

### 6.2 兼容策略

- 新增字段提供默认值
- 删除字段保留迁移映射
- 更名字段保留别名映射
- 废弃字段保留读取兼容期

### 6.3 破坏性变更

以下变更视为高风险：

- 删除实体 ID
- 更改核心引用关系
- 拆分关键字段结构
- 改变存档依赖字段

这类变更必须先给迁移方案，再允许发布。

## 7. 热重载与运行时读取

### 7.1 运行时原则

- 游戏默认读取发布包
- 游戏不直接读取后台草稿
- 运行时只保留必要缓存

### 7.2 重载范围

可考虑支持局部重载的内容：

- 文案
- 掉落表
- 部分数值参数
- UI 配置

不建议随意热重载的内容：

- 核心战斗逻辑
- 关卡结构
- 存档 schema
- 影响当前局的关键实体结构

## 7.3 首页展示配置

首页展示入口属于 `control/` 层配置，负责控制玩家首页看到哪些角色、怪物和 BOSS 预览。

可配置项包括：

- 是否开启首页预览入口
- 入口名称、入口图标和入口排序
- 展示实体 ID 列表
- 展示实体类型筛选
- 展示顺序和分组标签
- 是否公开可见
- 解锁规则
- 技能说明展示字段
- 预览动画、图标和特效引用

首页展示配置只负责“展示选择与展示方式”，不复制角色、怪物、BOSS 或技能本体配置；实体本体仍以全局或模块实体目录为准。

## 8. 调试与追踪

### 8.1 必须可追踪

每一局至少能追踪到：

- 使用了哪个发布版本
- 进入了哪个模块
- 使用了哪些角色、怪物、地图、技能配置
- 是否有局部覆盖生效
- 掉落和通关结果来自哪份配置

### 8.2 调试输出

建议提供：

- 当前生效配置打印
- 配置差异对比
- 引用链路追踪
- 缺失项报告
- 版本发布日志
- 首页展示实体引用报告

## 9. 模板与复制

### 9.1 新模块模板

新增模块时建议支持“从模板复制”：

- 模块总配置
- 基础角色目录
- 基础怪物目录
- 基础地图目录
- 基础技能目录
- 基础关卡节点组

### 9.2 模板价值

- 降低新模块创建成本
- 保持目录结构一致
- 减少漏配字段
- 便于后台快速落地

## 10. 权限与流程

### 10.1 推荐流程

1. 创建草稿
2. 选择目录和实体
3. 绑定素材
4. 自动校验
5. 预览确认
6. 发布
7. 归档历史版本

### 10.2 权限建议

- 内容编辑：可改草稿
- 审核人员：可校验和确认发布
- 发布人员：可生成正式发布包
- 管理员：可回滚和删除历史归档

## 11. 性能与规模

### 11.1 后台性能

当配置量变大时，需要支持：

- 索引检索
- 按模块过滤
- 按实体类型过滤
- 增量发布
- 缓存发布包

### 11.2 游戏侧性能

游戏侧需要：

- 启动时快速读取发布包
- 运行时按需加载素材
- 避免一次性载入全部资源

## 12. 设置系统配置

设置系统优先配置化，不把窗口、音量、键位和画质写死在代码里。

### 12.1 可配置项

- 窗口模式
- 分辨率
- 全屏 / 窗口化切换
- 主音量、音乐音量、音效音量、UI 音量
- 语言
- 键位映射
- 手柄映射
- 震动开关
- 字体大小
- UI 缩放
- 色弱模式
- 画面质量档位

### 12.2 配置方式

- 默认设置放在全局配置
- 模块可覆盖部分局部设置
- 玩家个人设置单独保存，不参与内容发布
- 后台负责维护默认档位和可选项集合

## 13. 音频系统配置

音频系统应通过配置表达，而不是在代码里写死音效名单。

### 13.1 可配置项

- 背景音乐列表
- 场景 BGM
- 战斗 BGM
- Boss BGM
- UI 音效
- 点击音效
- 命中音效
- 暴击音效
- 掉落音效
- 进入关卡音效
- 通关音效
- 失败音效
- 事件音效

### 13.2 音频规则

- 每个模块可配置独立音频包
- 每个实体可绑定自己的音效与触发点
- 支持同类音效随机池
- 支持音量分组和优先级
- 支持静音与淡入淡出参数

## 14. 经济与掉落配置

经济系统应配置化，避免金币、抽取概率和奖励曲线写死。

### 14.1 可配置项

- 金币掉落权重
- 道具抽取概率
- 普通 / 紫色 / 临时金色 / 永久金色权重
- 道具携带上限
- 道具抽取次数
- 关卡结算奖励
- 模块奖励曲线
- Boss 奖励池
- 通关稀有掉落概率

### 14.2 经济边界

- 经济参数应可按模块覆盖
- 不同难度可以使用不同权重表
- 永久资产和临时资产必须分开配置
- 后台应能看到配置后的预期收益区间

## 15. 随机种子与回放配置

为了让调试、复盘和数值验证可控，随机行为必须可配置。

### 15.1 可配置项

- 默认随机种子策略
- 是否使用固定种子
- 种子生成方式
- 每局种子记录方式
- 掉落随机池
- 怪物技能随机池
- 地图事件随机池

### 15.2 调试价值

- 同一局可复现
- Bug 可回放
- 掉落可对比
- 技能随机结果可追踪

## 16. 日志与追踪配置

日志不是临时代码，而应该是后台可定义的调试输出策略。

### 16.1 可配置项

- 日志级别
- 事件标签
- 记录范围
- 输出格式
- 是否落盘
- 是否导出调试包

### 16.2 应追踪事件

- 配置发布
- 配置加载
- 模块进入
- 关卡开始
- 关卡结算
- 掉落生成
- 存档写入
- 回滚生效

## 17. 无障碍与操作配置

无障碍和操作体验应尽量通过配置开放，而不是写死在界面里。

### 17.1 可配置项

- 字体大小
- 字幕开关
- 高对比度模式
- 色弱模式
- 文字描边强度
- 提示停留时长
- 输入提示显示方式
- 键位改绑
- 手柄提示样式

## 18. 统计与数据埋点配置

如果后续要做数据分析，也应通过配置定义埋点范围，而不是把统计点散落在各个系统里。

### 18.1 可配置项

- 关卡开始
- 关卡结束
- 通关失败
- 道具抽取
- Boss 击杀
- 模块解锁
- 存档写入
- 配置发布

### 18.2 作用

- 方便统计模块完成率
- 方便统计掉落体验
- 方便评估难度曲线
- 方便定位异常流失点

## 19. 调试与后台工具配置

调试入口和后台工具也尽量配置化，避免写成一堆临时硬编码按钮。

### 19.1 可配置项

- 指定模块启动
- 指定关卡启动
- 指定角色启动
- 指定道具组启动
- 指定存档状态启动
- 无敌模式
- 一键跳关
- 快速清怪
- 资源刷入
- 配置重载

### 19.2 原则

- 正式玩家默认不显示调试入口
- 开发环境和测试环境可以启用
- 调试入口应由后台或开发配置控制

## 20. 结论

全配置驱动不是单纯“把文件写成 YAML”，而是要同时把下面这条链路补齐：

`实体目录化 -> 稳定 ID -> 继承覆盖 -> 自动校验 -> 素材绑定 -> 预览确认 -> 发布版本 -> 运行时读取 -> 存档兼容 -> 调试追踪 -> 设置配置 -> 音频配置 -> 经济配置 -> 随机种子 -> 埋点配置 -> 调试工具`

这条链路完整后，新增一个模块、角色、怪物、地图或技能时，主要工作就会变成：

- 补目录
- 补配置
- 绑素材
- 过校验
- 发布版本

而不是改大量代码。
