# doc/7-验收

验收目录，负责沉淀实施前的验收标准、实施后的最终验收和放行结论。

## 当前入口

- 当前目录已建立，后续新增验收文档统一从这里进入。
- 当前仓库尚未形成正式验收标准或最终验收文档，后续以需求主干名持续补齐。

## 目录现状

- 当前仓库仍在需求、架构和实施规划阶段，尚未达到编写真正验收标准或最终验收结论的时点。
- 该目录继续保持有效占位，不提前伪造验收结论。

## 落点规则

- 验收标准命名：`<需求文档同名主干>验收标准.md`
- 最终验收命名：`<需求文档同名主干>最终验收.md`
- 前置验收标准与后置最终验收都不再混写进 `../2-需求/`、`../3-实施/` 或 `../5-tests/`。
- 同一需求的验收标准与最终验收默认持续更新原文件，不平行创建新的同主题验收入口。
- 后续一旦进入真实验收阶段，优先先补前置验收标准，再在测试与审查完成后补最终验收。
