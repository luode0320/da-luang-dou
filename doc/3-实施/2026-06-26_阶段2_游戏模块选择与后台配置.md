# 阶段 2：游戏模块选择 + 后台配置

## 阶段目标

阶段 2 负责游戏模块选择。玩家从首页进入模块地图或模块列表，看到不同主题模块、模块解锁状态、继续进度和开始新游戏入口。

后台侧必须能配置模块基础信息、模块展示、模块解锁、模块入口素材和模块状态。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 模块选择页 | 展示已发布模块、模块名称、主题图、描述、难度入口和解锁状态。 |
| 模块状态 | 支持未解锁、已解锁、进行中、已通关、归档隐藏。 |
| 继续上次 | 根据每模块单一自动存档进入上次进度。 |
| 开始新游戏 | 清除当前模块临时进度，从模块第一关开始。 |
| 模块入口 | 进入模块地图或关卡节点页。 |
| 范围边界 | 不做多存档槽，不做复杂世界地图动画，不做无尽模式入口。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 模块基础信息 | 模块 ID、中文名、简介、主题、版本、状态。 |
| 模块展示素材 | 模块封面、图标、背景、预览图。 |
| 模块解锁规则 | 默认解锁、依赖其他模块、依赖关卡通关、调试直接解锁。 |
| 模块难度入口 | 普通、噩梦、地狱是否开放。 |
| 模块发布状态 | 草稿、已发布、归档。 |

## 配置目录建议

```text
game/data/control/
└── modules_index.yaml                         # 模块索引和模块入口配置

game/data/modules/<module_id>/
├── module.yaml                                # 模块总配置
├── assets/                                    # 模块入口展示素材
└── publish/                                   # 模块发布产物
```

## 模块字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 模块稳定 ID。 |
| `name_zh` | 模块中文名。 |
| `description` | 模块介绍。 |
| `theme` | 模块主题。 |
| `status` | 草稿、已发布、归档。 |
| `cover_asset_id` | 模块封面。 |
| `unlock_rule_id` | 解锁规则。 |
| `difficulty_ids` | 开放难度列表。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-2-001 | 模块列表由后台配置，不在游戏代码写死模块。 |
| AC-2-002 | 游戏能展示模块解锁、进行中和已通关状态。 |
| AC-2-003 | 每个模块只保留一个自动存档状态。 |
| AC-2-004 | 后台能新建、复制、归档模块。 |
| AC-2-005 | 后台能配置模块入口素材、文案和解锁规则。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 阶段 1 首页入口、模块索引配置、自动存档基础设计。 |
| 输出给阶段 3 | 当前选中模块 ID、模块主题、模块地图配置入口。 |
