# 阶段 6：游戏关卡配置 + 后台配置

## 阶段目标

阶段 6 负责关卡配置和关卡推进。每个模块默认 20 关，每关默认 60 秒，关卡之间连续推进，不在每关结束后停住。

后台侧必须能配置关卡节点、关卡持续时间、刷怪波次、Boss 出现、奖励、难度和发布校验。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 关卡节点 | 模块内默认 20 关，按节点推进。 |
| 连续推进 | 当前关卡结束后默认进入下一关，不强制停住。 |
| 单关时长 | 每关默认 60 秒，可由配置覆盖。 |
| 背包暂停 | `I` 键打开背包，怪物和角色技能动作暂停。 |
| 局内抽道具 | 背包内可使用金币抽普通、紫色、临时金色道具。 |
| 通关保存 | 成功通关后写入关卡进度、模块解锁和奖励。 |
| 失败保护 | 失败不清除长期资产。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 关卡基础信息 | 关卡 ID、模块归属、关卡编号、节点类型、持续时间。 |
| 节点关系 | 前置节点、后续节点、解锁规则。 |
| 刷怪波次 | 出怪时间点、怪物类型、数量、位置、波次节奏。 |
| Boss 规则 | Boss 出现关卡、Boss 形态、狂暴概率。 |
| 难度配置 | 普通、噩梦、地狱规则映射。 |
| 奖励配置 | 金币、道具池、永久金色概率、通关奖励。 |
| 发布校验 | 20 关连续性、引用完整性、Boss 节点、素材缺失、难度映射。 |

## 配置目录建议

```text
game/data/modules/<module_id>/levels/<level_id>/
├── config.yaml                                # 单关配置
├── waves.yaml                                 # 刷怪波次
├── rewards.yaml                               # 奖励配置
└── overrides/                                 # 难度或模块覆盖

game/data/modules/<module_id>/spawns/<spawn_table_id>/
└── config.yaml                                # 刷怪表

game/data/modules/<module_id>/drops/<drop_table_id>/
└── config.yaml                                # 掉落表
```

## 关卡字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 关卡稳定 ID。 |
| `module_id` | 所属模块。 |
| `difficulty` | 难度。 |
| `level_index` | 关卡编号。 |
| `duration_sec` | 持续时间，默认 60 秒。 |
| `node_type` | 普通战斗、精英、Boss、事件。 |
| `enemy_wave_pack_id` | 刷怪波次。 |
| `boss_id` | Boss 引用。 |
| `reward_pack_id` | 奖励配置。 |
| `unlock_next_node_ids` | 解锁后续节点。 |
| `map_id` | 地图引用。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-6-001 | 每个模块默认 20 关结构完整。 |
| AC-6-002 | 每关默认 60 秒，持续时间由配置控制。 |
| AC-6-003 | 关卡之间可以连续推进。 |
| AC-6-004 | `I` 键背包暂停能暂停怪物和角色技能动作。 |
| AC-6-005 | 后台能配置刷怪时间点、怪物数量、Boss 出现和奖励。 |
| AC-6-006 | 发布前能校验关卡节点断链、引用缺失、难度映射和素材缺失。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 阶段 2 模块、阶段 3 地图、阶段 4 角色、阶段 5 怪物。 |
| 输出 | 完成首页 -> 模块选择 -> 地图 -> 角色 -> 怪物 -> 关卡战斗的第一版主链路。 |
