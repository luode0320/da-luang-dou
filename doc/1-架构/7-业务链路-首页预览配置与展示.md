# 业务链路-首页预览配置与展示

## 文档说明

- 业务目标: 描述首页角色、怪物和 BOSS 预览内容如何由后台配置，并在游戏首页按可见性与顺序展示。
- 入口: 后台首页配置入口；游戏首页预览展示入口
- 责任模块: 后台前端、后台后端、控制配置层、游戏运行层
- 事实来源: `项目设计.md`、`附录-后台系统结构方案.md`、`附录-配置目录规范.md`、`doc/2-需求/2026-06-23_215445_需求点总览v1.md`
- 最近同步时间: 2026-06-28 20:20:00
- 状态: 启用

## 前置条件

- 首页预览入口已在后台开启。
- 预览展示包、角色、怪物、BOSS 和技能说明已完成实体绑定。
- 相关素材和可见性状态已经通过后台校验。

## 主链路

1. 后台前端选择首页展示入口、排序、分组、可见性和推荐标记。
2. 后台后端将首页展示包与角色、怪物、BOSS、技能说明和预览素材绑定。
3. 发布时，首页展示配置写入 `game/data/control/` 对应入口。
4. 游戏首页读取展示入口配置，决定是否展示预览入口及其顺序。
5. 玩家进入预览展示后，游戏侧按已发布配置展示角色、怪物、BOSS 和技能说明。

## 异常分支

| 异常场景 | 处理方式 | 最终状态 |
| --- | --- | --- |
| 绑定实体不存在 | 后台校验失败并阻止发布 | 保持草稿状态 |
| 展示素材缺失 | 后台标记错误，游戏侧不静默兜底成假数据 | 阻断发布或隐藏异常项 |
| 配置为不可见 | 游戏首页不展示该入口或条目 | 不进入展示列表 |

## 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 首页展示入口配置、实体 ID、技能说明、图标与预览素材、解锁条件 |
| 输出 | 首页展示包、首页可见入口、展示顺序、玩家可见预览内容 |

## 变更影响

- 首页入口落点、默认可见性或展示包引用方式变化时，同步更新 [附录-首页预览展示包与首页入口规范](附录-首页预览展示包与首页入口规范.md) 与 [目录树](2-目录树.md)。
- 后台首页配置能力变化时，同步更新 [模块职责](3-模块职责.md) 和相关附录。
- 若首页预览不再属于顶层公共入口，需同步回写 [总架构](1-总架构.md) 与 `项目设计.md`。
