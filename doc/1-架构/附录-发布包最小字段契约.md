# 发布包最小字段契约

## 目标

本附录用于把“发布包至少要长什么样”写成最小可执行契约，避免后续后台发布器、Godot 读取器和测试验证各自脑补不同结构。

本附录解决四类问题：

- `manifest.yaml` 至少要有哪些字段
- `compatibility.yaml` 至少要有哪些字段
- `indexes/` 至少要提供哪些索引
- 游戏运行时应该先读什么、后读什么

## 适用范围

- `game/data/published/`
- 后台发布器
- 游戏配置读取器
- 阶段 0 配置驱动基础设施

| 主题 | 本文负责 | 应转交的专题 |
| --- | --- | --- |
| 发布包正文 | `manifest.yaml`、`compatibility.yaml`、`indexes/`、`control/`、`global/`、`modules/` 的最小目录与字段。 | 三态职责划分转交 [附录-发布包契约与三态边界](附录-发布包契约与三态边界.md)。 |
| 运行时读取顺序 | 运行时先读什么、后读什么、缺什么必须阻断。 | 版本兼容与回滚判定转交 [附录-版本矩阵与兼容判定规范](附录-版本矩阵与兼容判定规范.md)。 |

## 一、最小发布目录

长期建议至少满足以下最小目录结构：

```text
game/data/published/
└── <release_version>/
    ├── manifest.yaml
    ├── compatibility.yaml
    ├── indexes/
    │   ├── modules.yaml
    │   ├── entities.yaml
    │   └── assets.yaml
    ├── control/
    ├── global/
    └── modules/
```

说明：

- `control/`、`global/`、`modules/` 存放标准化后的配置正文
- `manifest.yaml` 提供版本与加载入口
- `compatibility.yaml` 提供兼容和迁移约束
- `indexes/` 提供正式索引入口与正文定位入口

## 二、manifest 最小字段

`manifest.yaml` 是游戏运行时进入发布包的第一入口。没有它，发布包不应被视为有效输入。

### 必填字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `release_version` | string | 当前发布版本唯一标识，默认与业务口径 `config_version` 一一对应 |
| `config_schema_version` | string | 当前发布包结构版本 |
| `created_at` | string | 发布时间，建议 ISO 8601 |
| `source_revision` | string | 后台草稿版本或构建来源标识 |
| `generator` | string | 生成器名称或发布器版本 |
| `default_module_id` | string | 默认模块入口 |
| `module_index_path` | string | `indexes/modules.yaml` 路径 |
| `entity_index_path` | string | `indexes/entities.yaml` 路径 |
| `asset_index_path` | string | `indexes/assets.yaml` 路径 |
| `compatibility_path` | string | `compatibility.yaml` 路径 |
| `content_hash` | string | 当前发布包摘要，用于完整性校验 |
| `status` | string | 例如 `published`、`rollback_target` |

### 推荐字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `previous_release_version` | string | 上一个正式版本 |
| `release_notes` | string | 当前发布摘要 |
| `warnings` | list | 非阻断发布警告 |
| `required_game_version` | string | 要求的最小游戏版本 |

### 示例

```yaml
release_version: release_2026_06_28_01
config_schema_version: v1
created_at: 2026-06-28T16:00:00+08:00
source_revision: draft_2026_06_28_120000
generator: admin-publisher-v1
default_module_id: module_forest
module_index_path: indexes/modules.yaml
entity_index_path: indexes/entities.yaml
asset_index_path: indexes/assets.yaml
compatibility_path: compatibility.yaml
content_hash: sha256:example
status: published
previous_release_version: release_2026_06_27_03
required_game_version: game_0.1.0
```

## 三、compatibility 最小字段

`compatibility.yaml` 是发布包中的兼容契约入口。本文只回答“它最少有哪些字段、位于哪里、读取器如何找到它”，不展开兼容判定规则本体。

### 必填字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `release_version` | string | 当前发布版本 |
| `min_supported_save_version` | string | 最低支持的存档版本 |
| `current_save_version` | string | 当前写入存档版本 |
| `field_aliases` | list | 字段更名兼容映射 |
| `removed_fields` | list | 已废弃字段说明 |
| `id_redirects` | list | 实体 ID 重定向映射 |
| `migration_notes` | list | 当前版本兼容说明 |

### 推荐字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `blocked_save_versions` | list | 明确禁止继续读取的历史存档版本 |
| `manual_repair_required` | list | 需要人工修复的数据风险 |
| `runtime_fallbacks` | list | 允许降级处理的字段引用 |

### 示例

```yaml
release_version: release_2026_06_28_01
min_supported_save_version: save_v1
current_save_version: save_v2
field_aliases:
  - from: display_name
    to: name_zh
removed_fields:
  - field: legacy_drop_rate
    replacement: drop_weight_map
id_redirects:
  - from: monster_slime_old
    to: monster_slime_green
migration_notes:
  - 新增掉落权重表，旧字段继续兼容一版
```

## 四、modules 索引最小字段

`indexes/modules.yaml` 用于让游戏和后台验证快速定位“有哪些模块、默认入口是什么、每个模块的正文在哪”。

### 必填字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `default_module_id` | string | 默认模块入口 |
| `modules` | list | 模块清单 |

### `modules` 子项最小字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `module_id` | string | 模块稳定 ID |
| `module_version` | string | 当前模块发布版本 |
| `name_zh` | string | 模块中文名 |
| `status` | string | 启用、禁用、归档等 |
| `module_path` | string | 模块正文路径 |
| `entry_level_id` | string | 默认入口关卡或节点 |
| `difficulty_pack_id` | string | 默认难度包 |

## 五、entities 索引最小字段

`indexes/entities.yaml` 用于快速定位实体，避免读取方必须先遍历完整正文目录才能定位实体正文。

### 必填字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `entities` | list | 实体索引清单 |

### `entities` 子项最小字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `entity_id` | string | 实体稳定 ID |
| `entity_type` | string | 角色、怪物、地图、技能、关卡、道具等 |
| `module_scope` | string | `global` 或具体 `module_id` |
| `body_path` | string | 实体正文路径 |
| `schema_version` | string | 实体结构版本 |
| `asset_refs` | list | 引用的素材 ID |
| `dependency_refs` | list | 引用的其他实体 ID |

## 六、assets 索引最小字段

`indexes/assets.yaml` 用于让后台发布器、读取方和预览系统快速知道“素材逻辑 ID 对应什么资源”。

### 必填字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `assets` | list | 素材索引清单 |

### `assets` 子项最小字段

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `asset_id` | string | 素材稳定 ID |
| `asset_version` | string | 当前素材发布版本 |
| `asset_type` | string | 图标、动画包、音频包、特效包等 |
| `logical_path` | string | 发布包内逻辑路径 |
| `source_entity_refs` | list | 引用该素材的实体 ID |
| `status` | string | 正式、占位、废弃 |

## 七、正文目录最小要求

### `control/`

至少覆盖：

- 首页入口
- 模块索引入口
- 当前发布元信息引用

### `global/`

至少覆盖：

- 公共实体默认值
- 公共规则
- 公共 schema 对应的已发布内容

### `modules/`

至少覆盖：

- 模块总配置
- 模块关卡与节点
- 模块局部覆盖

## 八、运行时读取顺序

游戏运行时推荐固定读取顺序：

1. 读取 `manifest.yaml`
2. 校验 `content_hash`、关键路径和版本字段
3. 读取 `compatibility.yaml`
4. 读取 `indexes/modules.yaml`
5. 读取 `indexes/entities.yaml`
6. 读取 `indexes/assets.yaml`
7. 进入目标模块后再按需读取 `control/`、`global/`、`modules/`
8. 把已定位到的正文入口交给后续运行时消费层

说明：

- 本文只固定“先读哪些正式文件与索引，再按需读取正文”的最小顺序。
- 读取后如何依据 `required_game_version`、`config_schema_version`、`field_aliases`、`blocked_save_versions` 做通过、警告或阻断，统一转交 [附录-版本矩阵与兼容判定规范](附录-版本矩阵与兼容判定规范.md)。
- 正文进入运行时后的合并、默认值、缓存、热重载和追踪策略，统一转交 [附录-游戏侧配置读取与合并规范](附录-游戏侧配置读取与合并规范.md)。

## 九、发布失败与读取失败的阻断条件

以下情况应视为阻断：

- 缺失 `manifest.yaml`
- `manifest.yaml` 缺必填字段
- 缺失 `compatibility.yaml`
- 缺失任一索引文件
- 默认模块入口不存在
- 索引中的正文路径不存在
- 正文引用实体或素材缺失
