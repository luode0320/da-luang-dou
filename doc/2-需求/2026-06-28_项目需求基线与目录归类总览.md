## 产品基线

本项目是一款面向 Steam/PC 的 2D 自动战斗闯关游戏。关卡组织参考《植物大战僵尸》的模块章节与节点推进，战斗表达参考《土豆兄弟》的地图内移动、自动攻击、持续清怪和局内成长节奏。

当前默认主循环不是无尽模式，而是模块化关卡选择制：

1. 启动游戏进入首页或模块入口。
2. 玩家进入模块地图或模块选择流程。
3. 读取当前模块唯一自动存档，决定继续上次或开始新游戏。
4. 选择模块关卡节点、角色和外部永久金色道具搭配。
5. 进入单关战斗，按连续关卡结构推进模块。
6. 结算通关、奖励、节点解锁、模块解锁和长期资产保留。

## P0 需求基线

| 领域 | 当前基线 |
| --- | --- |
| 主引擎 | 主游戏使用 Godot 4，不用 CSS/Web 承载主战斗、渲染、碰撞和动画。 |
| 平台 | 第一阶段也按 Steam/PC 桌面游戏交付，要求窗口、分辨率、键鼠输入、HUD、音画反馈、可导出构建和 Windows 安装链路。 |
| 主循环 | 默认进入模块地图或模块选择主流程，不以无尽模式作为第一入口。 |
| 模块结构 | 每个模块拥有独立地图主题、怪物生态、关卡节点、Boss 或模块终点。 |
| 关卡节奏 | 每模块默认 20 关，每关默认 60 秒，关卡之间连续推进。 |
| 暂停交互 | 玩家按 `I` 键打开背包即暂停，暂停期间怪物与角色技能动作停止。 |
| 难度 | 每模块默认普通、噩梦、地狱三档难度；难度主要由技能开放、Boss 阶段、地图压力和机制控制，不靠堆血量。 |
| 怪物分层 | 普通怪、精英怪、Boss 三层；Boss 支持基础、终极和狂暴形态。 |
| 角色 | 每个角色绑定固定武器和固定技能，不做角色技能树和武器切换。 |
| 道具 | 普通、紫色、临时金色属于模块内临时道具；永久金色属于长期外部资产，只能通过极低概率通关获得。 |
| 存档 | 每模块只保留一个自动存档；玩家只看到“继续上次”和“开始新游戏”。 |
| 失败保护 | 失败不清空模块解锁、已通关节点、永久金色和其他长期资产。 |
| 首页预览 | 首页提供角色、怪物、Boss、技能说明和动画预览的轻量入口，不等于完整图鉴。 |

## 配置与后台基线

| 领域 | 当前基线 |
| --- | --- |
| 配置真源 | 游戏内容以后台发布后的 YAML 配置包为默认真源，游戏运行时只读取发布结果。 |
| 目录结构 | `game/data/` 采用 `control/`、`global/`、`modules/<module_id>/` 三层结构。 |
| 后台系统 | `admin/` 拆分为 Go 后端与 Vue 前端，负责配置编辑、校验、预览、发布和回滚。 |
| 零硬编码 | 新增模块、角色、技能、怪物、地图、关卡、掉落规则时，原则上只改配置，不改业务代码。 |
| 发布边界 | 后台编辑态、发布契约态、游戏运行态三层分离，游戏不直接读取后台草稿。 |
| 字段与命名 | 中文用于展示，稳定 ID 用于系统引用、版本兼容和存档；文档和配置统一 UTF-8。 |

## 素材与运行时基线

| 领域 | 当前基线 |
| --- | --- |
| 素材流程 | 正式 2D 素材先通过 `2d-asset-design` / `imagegen` 完成设计预览，再进入 Godot 构建与接入。 |
| 地图素材 | 地图不只是一张背景图，至少要具备底图、props、碰撞和区域层。 |
| 运行时加速 | 第一版以 1x、2x、3x 三档循环为当前基线，统一影响公共时间、冷却、刷怪和掉落节奏。 |
| 错误处理 | 缺配置、缺素材、引用错误必须明确报错，至少标出实体 ID、模块 ID 和来源路径。 |
| 调试验证 | 开发和测试环境需要支持关卡直达、配置冒烟、存档恢复和回滚验证入口。 |

## 当前验收基线

| 编号 | 验收基线 |
| --- | --- |
| AC-001 | 首页、模块入口、模块节点、角色选择、战斗、结算和自动存档主链路可跑通。 |
| AC-002 | 至少一个模块能按默认 20 关、默认 60 秒结构持续推进。 |
| AC-003 | 普通、噩梦、地狱三档难度可以通过配置控制技能开放和 Boss 阶段。 |
| AC-004 | 游戏运行时只读取后台发布包，不直接依赖后台草稿。 |
| AC-005 | 新增模块、角色、怪物、技能、地图、关卡和掉落规则时，优先只改配置。 |
| AC-006 | Windows 版本可导出 exe 并具备安装器与快捷方式启动链路。 |
