# 架构目录README分组导航化收口

## 背景

- 在完成 `项目设计.md` 去专题化和 `1-总架构.md` 导航化后，`doc/1-架构/README.md` 仍然是超长链接清单。
- 如果目录 README 继续平铺所有文档链接，它虽然完整，但不利于快速判断“当前应该先看哪组入口”。

## 本轮处理结论

- 已将 `doc/1-架构/README.md` 收紧为分组导航页。
- 当前保留的核心结构为：
  - 先看哪一组
  - 稳定主入口
  - 业务链路入口
  - 阶段 0 与导览入口
  - 高重叠专题速查
  - 其他专题入口
  - 历史专题承接映射
  - 最小落点规则
- 已保留旧 `2026-06-22_*` 架构文档到正式附录的承接映射。
- 已保留控制配置层三件套、战斗载体三件套和发布契约层六件套的速查矩阵。

## 结果

- `doc/1-架构/README.md` 现在更像“架构目录首页”，而不是“所有文档平铺清单”。
- 后续再增加专题时，优先补到对应分组，不再直接把清单越拉越长。
