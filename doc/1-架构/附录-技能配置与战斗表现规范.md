# 技能配置与战斗表现规范

## 文档说明

- 用途: 统一项目中技能实体、角色固定技能、怪物基础技能、终极技能、Boss 阶段技能、技能特效、投射物、冷却规则和展示摘要的长期契约。
- 事实来源: `项目设计.md`、`doc/2-需求/2026-06-23_215445_需求点总览v1.md`、`doc/3-实施/2026-06-26_阶段4_GodotAI角色技能动作构建与后台配置游戏模块角色.md`、`doc/3-实施/2026-06-26_阶段5_GodotAI怪物技能动作构建与后台配置游戏模块怪物.md`、`doc/1-架构/附录-后台字段级配置规范.md`、`doc/1-架构/附录-配置目录规范.md`、`doc/1-架构/附录-游戏侧配置读取与合并规范.md`
- 适用范围: `game/data/global/skills/`、`game/data/modules/<module_id>/skills/`、`game/data/global/vfx/`、`game/data/modules/<module_id>/vfx/`、`game/data/global/characters/`、`game/data/global/enemies/`、`game/data/global/bosses/`、首页技能说明展示、运行时技能读取与发布校验
- 最近同步时间: 2026-06-28

## 分层总原则

### 1. 技能正文、实体绑定和预览展示必须分层

- 技能正文负责技能行为、冷却、目标规则、施法表现，以及技能如何引用投射物与命中效果。
- 角色、怪物和 Boss 正文只负责“绑定哪些技能”，不重复声明技能本体。
- 首页预览和技能说明展示只负责摘要展示，不承载技能真实行为与数值真源。

### 2. 技能实体是跨角色、跨怪物、跨模块复用的长期配置单元

- 技能优先作为独立实体目录管理，而不是散写在角色或怪物正文里。
- 模块技能覆盖允许调整模块特化表现，但不应破坏技能稳定 ID 与基础行为边界。
- 后续新增角色、怪物或 Boss 时，优先复用已有技能实体或新增技能实体，不通过硬编码直接写行为分支。

### 3. 战斗表现必须拆成施法、飞行、命中和冷却四段语义

- 施法阶段负责动作、前摇、施法特效和施法事件位点。
- 飞行阶段负责投射物、轨迹、速度和持续时间。
- 命中阶段负责伤害、状态效果入口、击退和命中特效。
- 冷却阶段负责技能再次可用时间和冷却展示，不与施法时长混写。
- 投射物包正文与命中反馈载体由 `附录-武器投射物与命中效果规范.md` 统一维护。
- Buff、Debuff、DOT、HOT、叠层与驱散规则由 `附录-状态效果与属性修饰规范.md` 统一维护。

### 4. 角色固定技能与怪物技能分层必须服从实体职责

- 角色正文通过 `fixed_skill_ids` 绑定固定技能。
- 普通怪、精英怪和 Boss 通过 `base_skill_ids`、`ultimate_skill_ids`、`phase_ids` 绑定技能层级。
- 技能升级树、自由技能池和角色运行时换技能不属于第一版范围。

## 分层边界速查

| 载体 | 本文负责 | 应转交的专题 |
| --- | --- | --- |

## 技能主配置最小契约

### `skills/<skill_id>/config.yaml`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `skill_id` | string | 技能稳定 ID。 |
| `name_zh` | string | 技能中文名。 |
| `skill_type` | string | `active` / `passive` / `triggered` / `ultimate`。 |
| `target_rule` | string | 单体、范围、自身、方向、区域等目标规则。 |
| `cast_time_sec` | number | 施法时间。 |
| `cooldown_sec` | number | 冷却时间。 |
| `range` | number | 技能作用距离。 |
| `damage_rule` | object | 伤害或效果规则。 |
| `hit_rule` | object | 命中规则。 |
| `animation_pack_id` | string | 技能动作或表现动画包。 |
| `vfx_pack_id` | string | 技能特效包。 |
| `sfx_pack_id` | string | 技能音效包。 |
| `projectile_pack_id` | string | 投射物配置包。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 主配置最小约束

- `skill_id`、`skill_type`、`target_rule`、`cooldown_sec`、`hit_rule` 缺失时不得发布。
- `ultimate` 技能必须显式标识为高阶技能，不得和基础技能共用模糊类型。
- `status=disabled` 的技能不得被正式角色、怪物或 Boss 配置引用。
- 技能稳定 ID 不因中文名、模块主题或视觉包装变化而改变。

## 施法、飞行、命中与冷却契约

### 施法阶段

建议至少支持：

- `cast_animation_state`
- `cast_event_markers`
- `cast_vfx_ref`
- `cast_time_sec`
- `interrupt_rule`

### 飞行阶段

建议至少支持：

- `projectile_type`
- `travel_speed`
- `travel_duration_sec`
- `collision_rule`
- `travel_vfx_ref`

### 命中阶段

建议至少支持：

- `damage_formula`
- `effect_refs`
- `knockback_rule`
- `hit_vfx_ref`
- `hit_sfx_ref`

### 冷却阶段

建议至少支持：

- `cooldown_sec`
- `cooldown_display_rule`
- `shared_cooldown_group`
- `cooldown_reset_rule`

## 角色固定技能绑定契约

### 绑定规则

- 每个角色通过 `fixed_skill_ids` 绑定固定技能列表。
- `fixed_skill_ids` 至少包含 1 个合法技能 ID。
- 角色固定技能优先绑定到全局技能实体；模块内确有特化时，通过模块技能覆盖承接。
- 角色固定技能不等于角色可自由编辑技能池。

### 与角色正文的边界

- 角色正文负责“这个角色带哪些技能”。
- 技能正文负责“这些技能如何施放、如何命中、如何冷却、如何展示”。
- 角色动画包负责角色动作本体；技能表现动画与特效由技能正文或表现资源承接。

## 怪物与 Boss 技能层级契约

### 普通怪与精英怪

- 普通怪只允许绑定最多 3 个基础技能。
- 精英怪允许绑定 3 个基础技能 + 1 个终极技能。
- `base_skill_ids` 与 `ultimate_skill_ids` 都必须引用合法技能实体。

### Boss 阶段技能

- Boss 主实体通过 `phase_ids` 组织阶段。
- 每个阶段单独声明可用 `base_skill_ids` 与 `ultimate_skill_ids`。
- 狂暴形态若复用旧技能，也应通过阶段配置显式声明，而不是运行时隐式拼接。

## 技能展示摘要契约

### 首页和预览入口

- 首页展示包中的技能说明只读取技能摘要字段，不读取完整战斗执行细节。
- 首页技能摘要建议最少包含 `name_zh`、`summary_zh`、`skill_type`、`preview_icon_asset_id`。
- 首页展示可引用技能说明和动作预览，但技能真实行为仍以技能正文为准。

### 后台预览

- 后台技能编辑器应至少能预览技能名称、类型、冷却、命中摘要、特效预览和投射物预览。
- 后台预览不应把技能说明字段和战斗行为字段混成单一输入面板。

## 运行时读取与缓存边界

### 读取时机

- 游戏启动时加载技能索引。
- 进入模块时加载模块覆盖技能。
- 进入关卡时加载当前角色、怪物和 Boss 依赖的技能正文与表现资源。

### 缓存边界

- 技能正文缓存与角色、怪物、Boss 缓存解耦。
- 技能 VFX、SFX、投射物配置可单独缓存。
- 不在每次施法时重新全量解析技能配置。

## 发布前最小校验集合

| 校验项 | 通过条件 | 阻断条件 |
| --- | --- | --- |
| 技能主配置完整性 | `skill_id`、类型、目标规则、冷却、命中规则齐全 | 关键字段缺失 |
| 角色固定技能合法性 | `fixed_skill_ids` 全部可解析 | 角色引用不存在技能 |
| 怪物技能分层合法性 | 普通怪、精英怪、Boss 技能数量和层级符合规则 | 普通怪挂终极技能、Boss 阶段技能缺失或越界 |
| 技能表现资源合法性 | 动画、特效、音效、投射物引用可解析 | 表现资源断链 |
| 首页技能摘要合法性 | 首页预览只读取技能摘要字段 | 首页展示直接依赖技能执行细节或引用断链 |
| 冷却口径合法性 | 冷却字段与阶段口径一致 | 把施法时长、冷却时间和触发间隔混写 |

## 结论

- 技能附录只维护技能实体本体、施法/飞行/命中/冷却语义和展示摘要。
- 投射物包、命中瞬时反馈和状态效果生命周期继续转交相邻专题，不在本文重复展开。

## 本文不负责的事项

- 不负责 `fixed_weapon_id` 与固定武器正文。
- 不负责 `projectiles/<projectile_pack_id>/` 的正文结构和命中瞬时反馈字段。
- 不负责 `effects/<effect_id>/` 的生命周期、叠层、驱散和清理规则。
