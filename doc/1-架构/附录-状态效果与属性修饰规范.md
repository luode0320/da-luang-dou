# 状态效果与属性修饰规范

## 文档说明

- 用途: 统一项目中 Buff、Debuff、持续伤害、持续治疗、临时属性增减、驱散、叠层、刷新与清理规则的长期契约。
- 事实来源: `项目设计.md`、`doc/1-架构/附录-武器投射物与命中效果规范.md`、`doc/1-架构/附录-技能配置与战斗表现规范.md`、`doc/1-架构/附录-道具分层与掉落奖励规范.md`、`doc/1-架构/附录-后台字段级配置规范.md`、`doc/1-架构/附录-配置目录规范.md`
- 适用范围: `game/data/global/effects/`、`game/data/modules/<module_id>/effects/`、`effect_ref`、`effect_refs`、`duration_type`、`tick_interval_sec`、`stack_rule`、`payload_ref`、`clear_rule`、运行时状态效果读取与发布校验
- 最近同步时间: 2026-06-28

## 分层总原则

### 1. 状态效果正文必须独立于武器、技能、道具和角色正文

- 武器正文负责“如何攻击、如何命中、命中时引用哪些效果包”。
- 技能正文负责“技能何时施放、如何引用效果包”。
- 道具正文负责“这个道具引用哪个效果包、属于哪种层级”。
- 效果包正文负责“效果本体是什么、持续多久、如何叠层、如何驱散、何时清理”。

### 2. 瞬时命中反馈与持续状态效果必须分层

- 瞬时命中反馈负责伤害瞬时结算、命中特效、命中音效和击退。
- 持续状态效果负责 Buff、Debuff、DOT、HOT、临时属性增减、持续触发和到期清理。
- 不允许把 DOT / HOT、叠层规则或驱散规则内联写进 `hit_rule`、道具说明文本或角色正文。

### 3. 状态效果是跨武器、跨技能、跨道具、跨模块复用的公共载体

- 同一 `effect_id` 可以被武器命中、技能施法、投射物超时、道具生效或关卡机制复用。
- 复用只通过 `effect_ref` 或 `effect_refs` 发生，不因来源不同复制第二套正文。
- 模块覆盖允许调整效果参数或表现，但不得破坏效果稳定 ID 与基础语义。

### 4. 状态效果要先定义“效果类型”，再定义“载荷”和“生命周期”

- 效果类型负责说明这是 Buff、Debuff、DOT、HOT、瞬时伤害还是纯属性修饰。
- 载荷负责说明具体修改哪些属性、造成何种持续结算或附加控制。
- 生命周期负责说明持续多久、如何叠层、如何刷新、何时驱散和何时清理。

## 分层边界速查

| 载体 | 本文负责 | 应转交的专题 |
| --- | --- | --- |
| `effects/<effect_id>/config.yaml` | 效果类型、载荷引用、生命周期、叠层、驱散与清理。 | 武器命中入口和技能施法入口分别转交武器 / 技能附录。 |
| `payloads/<payload_id>.yaml` | 属性修饰、DOT/HOT、控制效果等可执行载荷。 | 道具分层、奖励池和掉落来源转交 [附录-道具分层与掉落奖励规范](附录-道具分层与掉落奖励规范.md)。 |
| `effect_ref` / `effect_refs` | 只作为引用点，不在此重新定义来源武器、来源技能或来源道具。 | 来源实体如何绑定这些引用，转交对应正文附录。 |

## 目录与载体

### 全局默认效果包目录

```text
game/data/global/effects/<effect_id>/
├── config.yaml
├── payloads/
├── assets/
└── overrides/
```

### 模块覆盖目录

```text
game/data/modules/<module_id>/effects/<effect_id>/
├── config.yaml
├── payloads/
├── assets/
└── overrides/
```

### 与其他载体的边界

- `weapons/<weapon_id>/config.yaml` 只通过 `effect_refs` 引用效果包。
- `skills/<skill_id>/config.yaml` 或相关命中阶段配置只通过 `effect_refs` 引用效果包。
- `items/<item_id>/config.yaml` 只通过 `effect_ref` 引用效果包，不在道具正文中复制完整效果字段。
- `drops/`、`rewards.yaml`、`loadout_rules.yaml` 不定义效果包正文，只引用道具或技能结果。

## 效果包主配置最小契约

### `effects/<effect_id>/config.yaml`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `effect_id` | string | 效果包稳定 ID。 |
| `name_zh` | string | 效果中文名。 |
| `effect_type` | string | `buff` / `debuff` / `dot` / `hot` / `stat_modifier` / `instant_damage` / `control`。 |
| `apply_phase` | string | `on_hit` / `on_cast` / `on_expire` / `on_pickup` / `over_time`。 |
| `target_scope` | string | 目标范围或目标过滤规则。 |
| `duration_type` | string | `instant` / `timed` / `stacked` / `permanent_until_exit`。 |
| `duration_sec` | number | 持续时间。 |
| `tick_interval_sec` | number | 周期性结算间隔。 |
| `stack_rule` | object | 叠层、刷新、覆盖与上限规则。 |
| `payload_ref` | string | 实际属性或行为载荷引用。 |
| `clear_rule` | object | 到期、驱散、死亡、切图或结算时清理规则。 |
| `dispel_tags` | string[] | 可被哪些驱散类型移除。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 最小约束

- `effect_id`、`effect_type`、`apply_phase`、`duration_type`、`payload_ref` 缺失时不得发布。
- `duration_type=timed`、`stacked` 或 `permanent_until_exit` 时必须显式声明生命周期语义，不允许靠默认值猜测。
- `status=disabled` 的效果包不得被武器、技能、道具或关卡机制正式引用。
- 效果稳定 ID 不因来源武器、来源技能、道具名字或模块主题变化而改变。

## 效果载荷与属性修饰契约

### `payloads/<payload_id>.yaml` 建议字段

| 字段 | 说明 |
| --- | --- |
| `payload_id` | 载荷稳定 ID。 |
| `payload_type` | `stat_add` / `stat_mul` / `periodic_damage` / `periodic_heal` / `shield` / `slow` / `stun` / `silence` / `mark`。 |
| `target_stat` | 影响的目标属性，如攻速、移速、防御、技能冷却、生命回复。 |
| `base_value` | 基础数值。 |
| `ratio_value` | 比例数值。 |
| `tick_formula` | DOT / HOT 的单次结算公式。 |
| `max_cap_rule` | 上限规则。 |
| `source_snapshot_rule` | 是否在施加时快照来源属性。 |

### 最小约束

- 属性修饰必须显式声明修改的是哪一个运行时属性，不允许用模糊文案代替可执行载荷。
- DOT / HOT 必须显式声明 `tick_interval_sec` 与 `tick_formula` 的对应关系。
- 同时修改多个属性时，应通过可枚举载荷集合表达，不把多种效果混成单字符串描述。

## 生命周期、叠层与刷新契约

### `duration_type` 语义

- `instant`: 瞬时生效，无持续存在。
- `timed`: 固定时长持续，到期自动清理。
- `stacked`: 支持叠层，需显式定义叠层上限和刷新规则。
- `permanent_until_exit`: 当前模块或当前局内长期存在，直到退出或明确结算才清理。

### `stack_rule` 至少应支持

- `stack_mode`: `replace` / `refresh` / `additive` / `independent`
- `max_stacks`: 最大叠层数
- `refresh_on_reapply`: 重复施加时是否刷新持续时间
- `overflow_rule`: 超出上限时覆盖、延长、忽略还是转化

### 最小约束

- 若效果可叠层，必须显式声明 `stack_mode` 与 `max_stacks`。
- 若效果不可叠层，也应显式声明是 `replace` 还是 `refresh`。
- 叠层规则不得散落在道具掉落表、技能摘要或角色说明中。

## 驱散、清理与结算契约

### `clear_rule` 至少应支持

- `clear_on_death`
- `clear_on_stage_exit`
- `clear_on_module_exit`
- `clear_on_dispel`
- `clear_on_save_restore_blocked`
- `clear_on_stack_overflow`

### `dispel_tags` 建议语义

- `positive`
- `negative`
- `control`
- `timed`
- `module_temporary`

### 最小约束

- 驱散规则必须区分正面效果和负面效果，不允许所有状态统一清理。
- 模块内临时状态效果与长期外部资产效果的清理边界必须不同。
- 若某效果在存档恢复时不可保留，应显式在 `clear_rule` 中声明，而不是靠读取器隐式忽略。

## 与武器、技能、道具和存档的边界

- 武器附录和技能附录只负责 `effect_ref`、`effect_refs` 的引用入口，不负责效果正文。
- 道具附录只负责道具分层、获取方式、掉落表和长期 / 临时道具边界，不负责效果生命周期。
- 存档附录只负责状态效果是否进入恢复流程以及恢复兼容边界；本文只负责哪些效果会持续、叠层、驱散和清理。

## 运行时读取与缓存边界

### 读取时机

- 游戏启动时加载效果包索引与载荷索引。
- 进入模块时加载模块级 `effects/` 覆盖。
- 进入关卡后按角色武器、技能、怪物技能、道具和地图机制依赖加载实际需要的效果包正文。

### 缓存边界

- 效果包正文缓存与武器、技能、道具正文缓存解耦。
- 载荷配置可单独缓存，避免每次 DOT / HOT tick 重新解析配置。
- 同一 `effect_id` 被多来源复用时，应共享正文缓存，只在运行态实例层区分来源与目标。

## 发布前最小校验集合

| 校验项 | 通过条件 | 阻断条件 |
| --- | --- | --- |
| 效果主配置完整性 | 类型、阶段、生命周期、载荷齐全 | 关键字段缺失 |
| 载荷合法性 | `payload_ref` 可解析且目标属性明确 | 载荷断链或属性目标不明 |
| DOT / HOT 合法性 | 周期间隔与结算公式齐全 | 有持续结算但无 tick 规则 |
| 叠层规则合法性 | `stack_mode`、`max_stacks` 与刷新规则清晰 | 叠层上限或刷新行为不明 |
| 驱散清理合法性 | `clear_rule` 与 `dispel_tags` 语义一致 | 正负状态混清理或恢复边界不明 |
| 引用完整性 | `effect_ref`、`effect_refs` 全部可解析 | 武器、技能、道具引用不存在效果 |
| 模块覆盖合法性 | 模块覆盖不改稳定 ID，只改允许覆盖字段 | 模块覆盖直接改写效果边界 |

## 结论

- 状态效果附录只维护 `effects/<effect_id>/` 正文、属性载荷、生命周期、叠层刷新和驱散清理。
- 命中入口、技能入口、道具来源和存档恢复边界继续引用相邻专题，不在本文重复展开。

## 本文不负责的事项

- 不负责固定武器攻击节奏、投射物正文和命中瞬时反馈字段。
- 不负责技能本体冷却、施法阶段和展示摘要字段。
- 不负责道具层级、掉落表、奖励池和外部永久金色来源规则。
