# 主要业务链路索引与单链路正文去重收口

## 背景

- 在完成总架构、目录树和模块职责收口后，主链路层仍存在下一层重叠：
  - `4-主要业务链路.md` 既在做索引，也在替单链路正文解释链路职责边界。
  - `6-业务链路-模块闯关与进度推进.md`、`7-业务链路-首页预览配置与展示.md`、`8-业务链路-首页设置与运行时操作.md` 的结构还没有完全对齐 `5` 和 `9` 两条链路。
- 这会导致“索引页摘要”和“正文边界说明”同时存在两处，后续再优化时难以判断应该改哪一份。

## 本轮收口目标

- 让 `4-主要业务链路.md` 只承担“主链路索引页”职责。
- 让 `5-9` 各单链路文档承担“主流程、异常分支、输入输出、变更影响、本文不负责事项”职责。
- 让字段级、版本级、存档级、放行级说明继续回到对应附录。

## 本轮调整结果

### `4-主要业务链路.md`

- 新增“文档说明”“本文负责什么”“本文不负责什么”。
- 新增“阅读入口建议”，把“先看哪条链路”显式写出来。
- 将原“链路与专题分工提醒”改写成“索引页与单链路分工提醒”。
- 收紧后只保留：
  - 活动链路表
  - 阅读入口建议
  - 新增与更新规则
  - 索引页边界提醒

### `6-业务链路-模块闯关与进度推进.md`

- 新增“分层边界速查”。
- 明确把：
  - 模块索引与启动入口字段
  - 关卡节点与奖励字段
  - 存档与资产生命周期字段
  - 地图 / 角色 / 怪物 / 技能正文
  统一转交到各自附录。
- 新增“本文不负责的事项”。

### `7-业务链路-首页预览配置与展示.md`

- 新增“分层边界速查”。
- 明确把：
  - 首页展示包 / 公共导航 / 剧透等级字段
  - 角色 / 怪物 / Boss / 技能摘要正文
  - 发布契约与版本矩阵
  - 完整图鉴与收集系统
  转交到相邻附录或明确排除。
- 新增“本文不负责的事项”。

### `8-业务链路-首页设置与运行时操作.md`

- 新增“分层边界速查”。
- 明确把：
  - 默认设置 / 个人设置 / 语言 / 键位 / 音量 / 无障碍字段
  - 首页设置入口字段
  - 发布契约与持久化载体
  - UI 细节设计稿
  转交到相邻附录或明确排除。
- 新增“本文不负责的事项”。

## 当前分工结论

| 文档 | 现在主要回答的问题 | 不再回答的问题 |
| --- | --- | --- |
| `4-主要业务链路.md` | 当前有哪些主链路、各自入口是什么、应该先读哪一条 | 任一链路的主流程、异常分支、字段契约和专题正文 |
| `5-9` 单链路文档 | 某条链路如何流转、有哪些异常分支、输入输出是什么、变化会影响什么 | 版本矩阵、字段契约、记录载体、存档字段、放行证据等附录正文 |

## 后续维护规则

- 新增链路或废弃链路，优先更新 `4-主要业务链路.md` 的索引表与阅读入口。
- 链路流程、异常、输入输出和变更影响变化时，优先更新对应单链路文档。
- 如果变化下沉到字段、目录、记录载体、存档结构或放行证据，优先更新对应附录，不回到索引页重复写摘要。

## 结论

- 本轮进一步把主链路层拆成“索引真相源”和“单链路真相源”两层。
- 收口后：
  - `4-主要业务链路.md` 更接近“阅读入口和索引页”
  - `5-9` 更接近“主流程正文页”
- 后续若主链路层再次膨胀，默认继续按“索引问题回 4，流程问题回单链路，字段问题回附录”的规则收紧。
