# 业务链路-模块闯关与进度推进

- 业务目标: 描述玩家从首页进入模块地图，按模块章节和节点连续推进关卡，并保存单一自动存档进度的主流程。
- 入口: 游戏首页、模块地图、关卡节点入口
- 责任模块: 控制配置层、模块配置层、游戏运行层

## 主链路

1. 玩家启动游戏后进入首页。
2. 玩家在首页查看预览或直接进入模块地图。
3. 玩家选择继续上次或开始新游戏，进入模块章节和节点推进流程。
4. 游戏运行层在进入当前模块与关卡前，完成地图、实体、奖励和规则的正式配置装配。
5. 玩家进入单关后按当前关卡推进规则完成战斗，并在结算节点决定是否进入下一节点。
6. 运行时在模块推进过程中持续处理模块内临时状态、长期进度和恢复入口的分层写回。
7. 通关、失败、退出或重新开局时，运行时分别进入结算、保留、重置或恢复判断流程，再回到合法继续入口或默认起点。

## 异常分支

| 异常场景 | 处理方式 | 最终状态 |
| --- | --- | --- |
| 玩家失败退出当前关卡 | 进入失败结算与恢复入口判断 | 返回合法继续入口 |
| 当前模块配置缺失 | 游戏侧明确报错并阻断进入该模块 | 模块入口不可进入 |
| 关卡节点链路断裂 | 在校验或运行前标记问题节点 | 阻断发布或阻断进入 |
| `开始新游戏` 被错误实现为清空长期资产 | 视为实现错误，必须阻断上线 | 回退修正 |
