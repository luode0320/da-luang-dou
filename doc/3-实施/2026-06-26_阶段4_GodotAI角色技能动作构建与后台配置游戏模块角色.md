# 阶段 4：Godot AI MCP 角色 + 技能 + 动作构建 + 后台配置游戏模块角色

## 阶段目标

阶段 4 负责玩家角色。每个角色绑定固定武器和固定技能，不做角色技能升级树，不做武器切换。

长期固定武器、投射物、命中反馈和 Buff / Debuff 效果包的正文契约，统一以 `doc/1-架构/附录-武器投射物与命中效果规范.md` 为准；本文只保留阶段实施范围。

正式角色素材必须先通过 `imagegen` 设计，再通过 Godot AI MCP 接入角色动作、技能表现和资源绑定。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 角色实体 | 支持角色基础属性、移动、受击、死亡或失败表现。 |
| 固定武器 | 每个角色绑定固定武器，不提供武器切换。 |
| 固定技能 | 每个角色绑定固定技能，不做技能升级树。 |
| 动作动画 | 至少支持待机、移动、攻击、受击、死亡或失败动作。 |
| 角色选择 | 进入关卡前选择角色。 |
| 外部金色道具 | 进入关卡前搭配已拥有的外部金色道具。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 角色基础信息 | ID、中文名、说明、所属模块、标签。 |
| 武器绑定 | 固定武器 ID。 |
| 技能绑定 | 固定技能 ID 列表。 |
| 动作绑定 | 角色动作帧、动画包、帧率、循环规则。 |
| UI 素材 | 头像、图标、预览图。 |
| 解锁规则 | 默认可用、模块解锁、关卡解锁、调试可用。 |

## 配置目录建议

```text
game/data/modules/<module_id>/characters/<role_id>/
├── config.yaml                                # 角色主配置
├── assets/                                    # 头像、立绘、动作帧
├── skills/                                    # 模块角色技能覆盖
└── overrides/                                 # 模块局部覆盖
```

## 角色字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 角色稳定 ID。 |
| `name_zh` | 角色中文名。 |
| `fixed_weapon_id` | 固定武器。 |
| `fixed_skill_ids` | 固定技能列表。 |
| `portrait_asset_id` | 头像或立绘。 |
| `animation_pack_id` | 动作动画包。 |
| `ui_icon_asset_id` | UI 图标。 |
| `unlock_rule_id` | 解锁规则。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-4-001 | 角色由后台配置生成，不在游戏代码写死角色列表。 |
| AC-4-002 | 每个角色只能使用配置绑定的固定武器和固定技能。 |
| AC-4-003 | 角色动作素材先经 `imagegen` 设计，再由 Godot AI MCP 接入。 |
| AC-4-004 | 后台能预览角色动作帧、头像、技能说明。 |
| AC-4-005 | 进入关卡前能选择角色和外部金色道具。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 阶段 3 地图移动区域、角色出生点。 |
| 输出给阶段 5 | 怪物技能命中角色所需的角色受击和碰撞基础。 |
| 输出给阶段 6 | 关卡进入前的角色选择和道具搭配结果。 |
