# 武器投射物与命中效果规范

## 文档说明

- 用途: 统一项目中固定武器、投射物配置包、命中反馈以及它们与角色、技能、运行时读取之间的长期契约。
- 事实来源: `项目设计.md`、`doc/3-实施/2026-06-08_234500_模块化关卡最高级原则实施总览.md`、`doc/3-实施/2026-06-26_阶段4_GodotAI角色技能动作构建与后台配置游戏模块角色.md`、`doc/3-实施/2026-06-26_阶段5_GodotAI怪物技能动作构建与后台配置游戏模块怪物.md`、`doc/1-架构/附录-角色固定绑定与选择规范.md`、`doc/1-架构/附录-技能配置与战斗表现规范.md`、`doc/1-架构/附录-配置目录规范.md`、`doc/1-架构/附录-后台字段级配置规范.md`
- 适用范围: `game/data/global/weapons/`、`game/data/global/projectiles/`、`game/data/modules/<module_id>/weapons/`、`game/data/modules/<module_id>/projectiles/`、`fixed_weapon_id`、`projectile_pack_id`、`effect_refs`、运行时攻击读取与发布校验
- 最近同步时间: 2026-06-28

## 分层总原则

### 1. 角色绑定、武器正文、技能正文和效果正文必须分层

- 角色正文只负责通过 `fixed_weapon_id` 绑定固定武器，不重复声明武器本体、投射物和命中效果正文。
- 武器正文负责基础自动攻击形式、攻击节奏、命中入口和默认表现资源。
- 技能正文负责主动技能或触发技能自身的行为规则；当技能需要投射物、命中反馈或状态效果时，通过引用投射物 / 效果包契约复用，不在技能附录中重复定义这些载体正文。
- 状态效果正文由 `附录-状态效果与属性修饰规范.md` 统一维护，不在本文重复展开其生命周期与叠层细节。

### 2. 固定武器是第一版角色自动攻击入口，不是可自由切换的装备树

- 每个角色必须且只允许绑定 1 个固定武器。
- 第一版不支持关卡内武器切换，不支持自由武器树，不支持把模块内掉落道具直接替代角色固定武器。
- 后台可以替换 `fixed_weapon_id` 的绑定关系，但发布后的单局运行态仍只读取 1 个固定武器入口。

### 3. 投射物、命中反馈和持续效果必须拆成三层

- 投射物层负责生成、飞行、碰撞、穿透、反弹、寿命和销毁规则。
- 命中反馈层负责命中瞬间的伤害结算入口、命中特效、命中音效、击退和镜头反馈。
- 效果包层负责 Buff、Debuff、持续伤害、持续治疗、临时属性增减等持续型或可叠层效果。

### 4. 武器与技能可以共用投射物和效果包，但不能把载体正文混成一份

- 固定武器和主动技能都可以复用同一 `projectile_pack_id` 或 `effect_refs`。
- 复用只发生在“引用同一载体”层，不表示把固定武器正文和技能正文合并为同一配置实体。
- 同一投射物包或效果包的稳定 ID 不因绑定角色、技能来源或模块主题变化而改变。

### 5. 阶段文档只保留实施安排，长期武器规则统一回收架构层

- 阶段 4 只保留“如何接入角色和固定武器”的实施安排。
- 阶段 5 只保留“如何接入怪物技能和表现资源”的实施安排。
- 固定武器、投射物、命中反馈、Buff / Debuff 效果包的长期字段和目录契约以本文为准。

## 武器主配置最小契约

### `weapons/<weapon_id>/config.yaml`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `weapon_id` | string | 武器稳定 ID。 |
| `name_zh` | string | 武器中文名。 |
| `weapon_type` | string | `melee` / `projectile` / `beam` / `orbit` / `summon_entry`。 |
| `attack_interval_sec` | number | 自动攻击基础间隔。 |
| `aim_rule` | string | 自动瞄准、方向锁定、最近目标、前方扇形等规则。 |
| `damage_rule` | object | 武器命中时的基础伤害或倍率规则。 |
| `hit_rule` | object | 命中模式、碰撞数、穿透、击退等命中规则。 |
| `projectile_pack_id` | string | 默认投射物配置包；近战直击型可为空。 |
| `effect_refs` | string[] | 命中后附加的状态效果引用。 |
| `animation_pack_id` | string | 武器攻击或表现动画包。 |
| `vfx_pack_id` | string | 武器施放或轨迹特效包。 |
| `sfx_pack_id` | string | 武器施放或命中音效包。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 最小约束

- `weapon_id`、`weapon_type`、`attack_interval_sec`、`hit_rule` 缺失时不得发布。
- `weapon_type=projectile` 时必须声明合法 `projectile_pack_id`。
- `weapon_type=melee` 若不使用投射物，必须在 `hit_rule` 中显式声明直击命中模式。
- `status=disabled` 的武器不得被正式角色的 `fixed_weapon_id` 引用。
- 固定武器稳定 ID 不因角色皮肤、模块主题或投射物表现变化而改变。

## 投射物配置包最小契约

### `projectiles/<projectile_pack_id>/config.yaml`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `projectile_pack_id` | string | 投射物配置包稳定 ID。 |
| `projectile_type` | string | `bullet` / `arrow` / `beam_segment` / `orb` / `thrown`。 |
| `spawn_anchor` | string | 从角色、武器骨点、技能施法位点或怪物节点生成。 |
| `travel_speed` | number | 飞行速度。 |
| `lifetime_sec` | number | 存活时间上限。 |
| `max_distance` | number | 最大飞行距离。 |
| `collision_rule` | object | 命中单位、障碍、地形和区域的碰撞规则。 |
| `penetration_count` | number | 最大穿透数量。 |
| `bounce_rule` | object | 是否允许反弹及反弹次数规则。 |
| `expire_effect_ref` | string | 超时或落地时的结束效果包引用。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 最小约束

- 被武器或技能引用的 `projectile_pack_id` 必须能解析到合法投射物包。
- 投射物包不得在正文中硬编码所属角色、所属技能或所属怪物 ID。
- `lifetime_sec` 与 `max_distance` 至少应有一项可独立限制投射物无限存在。
- `status=disabled` 的投射物包不得被正式发布配置引用。

## 命中反馈契约

### 命中反馈层最小语义

`hit_rule` 至少应能表达以下语义：

- `hit_mode`: `direct` / `projectile_contact` / `beam_tick` / `aoe_pulse`
- `target_scope`: 单体、范围、多段、穿透后多目标
- `knockback_rule`: 是否击退、击退距离和方向
- `hit_vfx_ref`: 命中特效引用
- `hit_sfx_ref`: 命中音效引用
- `effect_refs`: 命中后施加的状态效果列表

### 最小约束

- `hit_rule` 应只表达命中触发入口、目标范围、击退与即时反馈，不承载状态效果生命周期正文。
- `effect_refs` 只引用状态效果稳定 ID；效果包正文、叠层、驱散与清理规则由 `附录-状态效果与属性修饰规范.md` 维护。

## 角色固定武器、技能引用与怪物边界

### 角色侧边界

- 角色附录只负责“哪个角色绑定哪个 `fixed_weapon_id`”。
- 固定武器正文决定角色的基础自动攻击形式、默认投射物入口和默认命中效果。
- 角色选择页输出 `fixed_weapon_id` 后，运行态再继续解析武器、投射物和效果包正文。

### 技能侧边界

- 技能附录负责技能实体本体、施法 / 冷却语义以及技能如何引用 `projectile_pack_id`、`effect_refs`。
- 投射物包正文和命中反馈载体规则由本文负责；状态效果正文由 `附录-状态效果与属性修饰规范.md` 负责。

### 怪物与 Boss 侧边界

- 普通怪、精英怪和 Boss 第一版主要通过技能层级表达攻击行为。
- 若某个怪物技能复用和角色固定武器相同的投射物或状态效果，仍通过 `projectile_pack_id`、`effect_refs` 引用，不新增一套怪物专属载体正文。
- 本文不引入“怪物自由武器树”；怪物与 Boss 的长期边界仍以技能附录和怪物 Boss 附录为准。

## 运行时读取与缓存边界

### 读取时机

- 游戏启动时读取武器与投射物索引。
- 进入模块时读取模块级 `weapons/`、`projectiles/` 覆盖。
- 进入关卡时按本次 `fixed_weapon_id`、`fixed_skill_ids` 和怪物技能依赖加载实际需要的武器与投射物正文，并通过效果系统索引补充 `effect_refs`。

### 缓存边界

- 武器正文缓存与角色正文缓存解耦。
- 投射物包可单独缓存，不在每次攻击时重新全量解析。
- 同一 `projectile_pack_id` 被多个武器或技能复用时，应共享缓存，而不是为每个来源复制一份运行时实例配置。

## 发布前最小校验集合

| 校验项 | 通过条件 | 阻断条件 |
| --- | --- | --- |
| 固定武器合法性 | `fixed_weapon_id` 可解析到合法武器正文 | 角色引用不存在武器或引用禁用武器 |
| 武器主配置完整性 | 武器类型、攻击间隔、命中规则齐全 | 关键字段缺失 |
| 投射物引用完整性 | `projectile_pack_id` 可解析且符合武器类型 | 远程武器无投射物包或引用断链 |
| 命中反馈合法性 | `hit_rule`、命中特效和命中音效引用可解析 | 命中载体缺失或语义冲突 |
| 状态效果引用合法性 | `effect_refs` 全部能解析到合法状态效果 | 状态效果断链或状态非法 |
| 模块覆盖合法性 | 模块覆盖不改稳定 ID，只改允许覆盖字段 | 模块覆盖直接改写实体边界 |
| 技能复用合法性 | 技能对投射物和效果包的引用与载体正文一致 | 技能正文内联第二套投射物或效果包结构 |

## 结论

- 武器附录只维护固定武器正文、投射物包和命中瞬时反馈三层载体。
- Buff、Debuff、DOT、HOT、叠层与驱散正文继续转交状态效果专题，不在本文重复维护。

## 本文不负责的事项

- 不负责技能实体本体、施法阶段语义和冷却展示规则。
- 不负责 Buff、Debuff、DOT、HOT、属性修饰的生命周期与叠层正文。
- 不负责道具分层、掉落表和奖励池规则。
