# 阶段 7：测试验证与发布收口

## 阶段目标

阶段 7 负责把前面阶段形成的内容闭环收成可验证、可发布、可回滚、可打包的 PC 游戏版本。它不是新增玩法阶段，而是保证配置驱动游戏可以稳定上线和持续迭代的收口阶段。

本阶段重点处理配置冒烟测试、关卡直达调试、存档兼容、发布回滚、Windows 打包、Steam/PC 基础设置和版本矩阵。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 配置冒烟启动 | 使用发布包启动游戏，确认首页、模块、地图、角色、怪物、关卡都能被读取。 |
| 关卡直达调试 | 开发和测试环境支持指定模块、关卡、难度、角色、道具和存档状态启动。 |
| 存档兼容验证 | 旧自动存档在新增模块、关卡、角色、道具和配置字段后仍能继续使用。 |
| 发布包回滚验证 | 回滚到上一发布包后，游戏能正常启动并读取对应配置。 |
| Windows 打包 | Godot 导出 Windows exe，并形成安装器和快捷方式流程。 |
| PC 设置验证 | 验证窗口、分辨率、音量、键位、暂停、加速和基础输入体验。 |
| 运行日志 | 输出配置版本、模块版本、关卡 ID、实体 ID、错误和警告。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| 发布前检查 | 校验模块完整性、20 关连续性、Boss 关卡、素材缺失、引用断链和难度映射。 |
| 冒烟测试任务 | 后台提供一键检查或导出测试清单，覆盖首页、模块、地图、角色、怪物、关卡。 |
| 版本矩阵 | 展示游戏版本、配置版本、模块版本、素材版本和存档版本。 |
| 发布记录 | 记录发布时间、发布人、变更摘要、校验结果和兼容说明。 |
| 回滚记录 | 记录回滚版本、回滚原因、影响模块和回滚验证结果。 |
| 权限审计 | 发布、回滚和删除等高风险操作必须记录操作者和时间。 |
| 打包参数 | 管理 Windows 构建版本号、导出渠道、发布说明和安装包元信息。 |

## 配置目录建议

```text
game/data/published/<publish_version>/
├── manifest.yaml                              # 发布包清单
├── checks.yaml                                # 发布前校验结果
├── compatibility.yaml                         # 存档兼容说明
└── modules/                                   # 发布后的模块配置快照

game/data/control/debug/
├── launch_profiles.yaml                       # 开发测试直达配置
└── smoke_tests.yaml                           # 配置冒烟测试清单

admin/releases/<release_id>/
├── summary.yaml                               # 发布摘要
├── rollback.yaml                              # 回滚信息
└── audit.yaml                                 # 操作审计
```

## 后台字段建议

| 字段 | 用途 |
| --- | --- |
| `release_id` | 发布记录稳定 ID。 |
| `game_version` | 对应游戏构建版本。 |
| `config_version` | 对应配置发布版本。 |
| `module_versions` | 本次包含的模块版本列表。 |
| `asset_versions` | 本次包含的素材版本列表。 |
| `save_schema_version` | 存档结构版本。 |
| `validation_status` | 发布前校验结果。 |
| `rollback_target_id` | 可回滚目标版本。 |
| `build_channel` | 开发、测试、正式等构建渠道。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-7-001 | 后台发布前能执行配置完整性冒烟检查。 |
| AC-7-002 | 游戏能通过发布包完成首页 -> 模块选择 -> 地图 -> 角色 -> 怪物 -> 关卡战斗链路验证。 |
| AC-7-003 | 开发测试环境支持关卡直达，不作为正式玩家入口。 |
| AC-7-004 | 旧自动存档在兼容范围内可继续读取。 |
| AC-7-005 | 发布包支持回滚，并能记录回滚原因和验证结果。 |
| AC-7-006 | Windows exe 和安装器流程可执行，启动后能进入游戏首页。 |
| AC-7-007 | 版本矩阵能追踪游戏、配置、模块、素材和存档版本。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 阶段 0 配置基础设施，以及阶段 1-6 的核心链路。 |
| 输出 | 可验证、可回滚、可打包的 PC Demo 发布闭环。 |
