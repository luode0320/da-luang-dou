# 角色固定绑定与选择规范

## 文档说明

- 用途: 统一项目中角色实体、固定武器、固定技能、角色选择输入、外部永久金色道具搭配、解锁规则和动作预览的长期契约。
- 事实来源: `项目设计.md`、`doc/3-实施/2026-06-26_阶段4_GodotAI角色技能动作构建与后台配置游戏模块角色.md`、`doc/1-架构/附录-配置目录规范.md`、`doc/1-架构/附录-后台字段级配置规范.md`、`doc/1-架构/附录-存档与资产生命周期规范.md`、`doc/1-架构/附录-模块索引与启动入口规范.md`
- 适用范围: `game/data/global/characters/`、`game/data/modules/<module_id>/characters/`、角色选择入口、外部永久金色道具搭配入口、角色发布校验
- 最近同步时间: 2026-06-28

## 分层总原则

### 1. 角色层只负责“玩家本局用谁”和“固定绑定什么”，不替代关卡、模块和掉落正文

- 角色正文负责角色基础属性、固定武器、固定技能、动作包、头像、说明、解锁规则和预览资源。
- 模块正文负责“当前模块允许使用哪些角色、默认推荐哪个角色、哪些角色在本模块可见”。
- 关卡入口负责“当前这次进入关卡最终选中了哪个角色、携带了哪些外部永久金色道具”。
- 存档层负责“当前模块最近一次使用的角色、当前外部永久金色道具搭配结果和已解锁角色列表”。

### 2. 角色固定绑定是长期玩法约束，不允许运行时切换为可变武器树

- 每个角色绑定固定武器，不提供关卡内武器切换。
- 每个角色绑定固定技能列表，不提供角色技能升级树。
- 后续新增角色时优先补角色配置和资源，不通过新增角色专属硬编码分支接入。

### 3. 角色选择入口与首页预览入口分层

- 首页预览入口只负责展示后台选择的角色预览、说明和动作，不承担实际进关选择结果。
- 模块入口或关卡入口中的角色选择页才产生本次运行态正式 `role_id`。
- 首页预览中展示的角色不要求等于当前模块可选角色全集，但必须引用合法角色实体。

## 角色主配置最小契约

### `config.yaml`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `role_id` | string | 角色稳定 ID。 |
| `name_zh` | string | 角色中文名。 |
| `summary_zh` | string | 角色简短说明。 |
| `role_tags` | string[] | 角色标签，如近战、远程、生存、召唤。 |
| `portrait_asset_id` | string | 头像或立绘资源引用。 |
| `ui_icon_asset_id` | string | UI 列表图标引用。 |
| `animation_pack_id` | string | 动作动画包引用。 |
| `fixed_weapon_id` | string | 固定武器 ID。 |
| `fixed_skill_ids` | string[] | 固定技能 ID 列表。 |
| `unlock_rule_id` | string | 解锁规则引用。 |
| `preview_visible` | bool | 是否允许出现在首页或预览入口。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 最小约束

- `role_id`、`fixed_weapon_id`、`fixed_skill_ids`、`animation_pack_id`、`unlock_rule_id` 缺任一项都不得发布。
- `fixed_skill_ids` 至少包含 1 个合法技能 ID。
- `status=disabled` 的角色不得出现在正式可选角色名单中。
- `status=deprecated` 的角色仅允许在历史回放、兼容恢复或后台迁移提示中保留引用。

## 固定武器与固定技能绑定契约

### 固定武器绑定

- 每个角色必须且只允许绑定 1 个固定武器。
- 固定武器负责角色的基础自动攻击形式、投射物或近战判定入口。
- 后台可以替换角色绑定的武器引用，但发布后的单个角色运行态仍只读取一个 `fixed_weapon_id`。
- 固定武器正文、投射物配置包和命中效果载体由 `附录-武器投射物与命中效果规范.md` 统一维护；本文只定义角色如何绑定 `fixed_weapon_id`。

### 固定技能绑定

- 每个角色默认绑定 1 组固定技能列表，运行态通过 `fixed_skill_ids` 直接读取。
- 角色技能不提供成长树，不允许通过升级在同一角色内切换到另一组完全不同的角色技能树。
- 后续若存在阶段性解锁技能，也应通过固定技能组内的条件启用表达，而不是把角色变成自由技能池。

## 角色选择输入契约

### 角色选择页最小输入

| 字段 | 说明 |
| --- | --- |
| `release_version` | 当前读取的发布版本。 |
| `module_id` | 当前准备进入的模块。 |
| `difficulty_id` | 当前选择的难度。 |
| `available_role_ids` | 当前模块、当前难度下可见且可选的角色列表。 |
| `default_role_id` | 默认推荐角色。 |
| `last_role_id` | 当前模块最近一次使用的角色。 |
| `unlocked_role_ids` | 当前玩家已解锁角色列表。 |

### 角色选择页最小输出

| 字段 | 说明 |
| --- | --- |
| `role_id` | 本次正式进入关卡的角色 ID。 |
| `fixed_weapon_id` | 运行态锁定的固定武器。 |
| `fixed_skill_ids` | 运行态锁定的固定技能列表。 |
| `selected_loadout_item_ids` | 本次进入关卡搭配的外部永久金色道具。 |
| `selection_source` | `default` / `manual` / `resume` / `debug_direct`。 |

## 外部永久金色道具搭配契约

### 分层原则

- 外部永久金色道具属于长期资产，不属于角色正文配置本体。
- 角色正文只约束“允许搭配外部永久金色道具”这一入口，不直接固化某个永久金色道具为角色本体字段。
- 本次进入关卡的外部永久金色道具搭配结果应作为角色选择输出的一部分进入运行态与存档层。

### `loadout_rules.yaml` 建议字段

| 字段 | 说明 |
| --- | --- |
| `module_id` | 所属模块。 |
| `max_external_gold_items` | 当前模块允许携带的外部永久金色道具数量上限。 |
| `allowed_item_tags` | 当前模块允许携带的永久金色道具标签范围。 |
| `blocked_item_ids` | 当前模块禁止搭配的永久金色道具列表。 |
| `default_loadout_item_ids` | 默认推荐搭配。 |

## 角色解锁规则契约

### `unlock_rule_id` 最小语义

- `default_open`：默认可用。
- `module_unlock`：解锁指定模块后可用。
- `node_unlock`：通关指定节点后可用。
- `achievement_unlock`：完成指定长期目标后可用。
- `debug_only`：仅调试和验证入口可见。

### 解锁状态最小输出

| 字段 | 说明 |
| --- | --- |
| `unlock_state` | `locked` / `available` / `selected` / `debug_only`。 |
| `unlock_reason` | 当前状态的来源说明。 |
| `unlock_progress_ref` | 与模块、节点或长期进度对应的引用。 |

## 动画、预览与资源绑定契约

### `animation_pack_id` 最小要求

- 至少支持待机、移动、攻击、受击、失败或死亡动作。
- 如果角色存在施法显式表现，动画包中还应提供施法动作或事件位点。
- 角色动作资源必须先经过 `imagegen` 设计预览，再进入 Godot AI MCP 构建与接入流程。

### `portrait_asset_id` 与 `ui_icon_asset_id`

- `portrait_asset_id` 用于角色选择页、首页预览和详情展示。
- `ui_icon_asset_id` 用于列表、卡片、状态条或模块入口的轻量引用。
- 二者可以指向同一资源，但运行态语义应分开保留，避免后续一处改资源影响全部展示场景。

## 发布前最小校验集合

| 校验项 | 通过条件 | 阻断条件 |
| --- | --- | --- |
| 角色主配置完整性 | `role_id`、武器、技能、动画、解锁规则齐全 | 关键字段缺失 |
| 固定绑定合法性 | 固定武器和固定技能均能解析到合法实体 | 武器或技能引用断链 |
| 模块角色名单合法性 | `available_role_ids` 都能解析到合法角色 | 模块引用角色不存在 |
| 解锁规则可解析 | `unlock_rule_id` 能映射到合法规则 | 规则缺失或状态不明 |
| 预览资源可解析 | 头像、图标、动画包引用存在 | 首页或选择页资源断链 |
| 存档恢复兼容性 | 历史 `role_id` 仍能恢复或可映射 | 历史角色已删除且无兼容映射 |
| 外部道具上限规则 | 角色选择输出与模块搭配规则一致 | 超上限或使用禁用永久金色道具 |

  - 调试直达可以指定 `role_id`；
  - `role_id` 的合法性和固定绑定边界仍以本文为准。

## 结论

- 角色长期不是“一个可自由拼装的技能容器”，而是“固定武器 + 固定技能 + 动作包 + 解锁规则 + 预览资源”的稳定配置单元。
- 角色正文、模块角色名单、外部永久金色道具搭配和存档恢复必须分层。
- 后续阶段 4 的后台配置器、Godot 角色选择流程、首页角色预览和调试入口都应直接复用本文，不再各自解释角色绑定边界。
