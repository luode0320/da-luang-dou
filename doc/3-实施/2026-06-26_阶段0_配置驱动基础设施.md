# 阶段 0：配置驱动基础设施

## 阶段目标

阶段 0 负责整个项目的配置底座。它不是一个玩家可见功能，而是后续首页、模块、地图、角色、怪物、关卡、道具、掉落、发布和存档都必须依赖的基础设施。

本阶段的核心目标是先把“能配置就不写死”落成可执行规则：YAML 结构、稳定 ID、目录规范、发布包、校验、合并、运行时加载、版本兼容和调试追踪必须先有统一口径。

## 游戏侧范围

| 模块 | 需求 |
| --- | --- |
| 配置加载器 | Godot 游戏侧只读取后台发布后的配置包，不直接读取后台草稿。 |
| 合并规则 | 支持全局默认、模块覆盖、实体覆盖和运行时默认值的稳定合并。 |
| 稳定 ID | 所有模块、角色、怪物、技能、地图、关卡、道具、掉落表、素材都使用稳定 ID 引用。 |
| 默认值 | 新增字段必须具备默认值或兼容处理，避免旧配置启动失败。 |
| 错误处理 | 缺配置、缺素材、引用断链、字段异常时明确报错并定位到配置路径。 |
| 缓存策略 | 启动缓存总索引，进入模块缓存模块配置，进入关卡缓存当前关卡依赖实体。 |
| 调试追踪 | 记录当前发布版本、模块 ID、关卡 ID、实体 ID、覆盖来源和警告项。 |

## 后台配置范围

| 配置项 | 说明 |
| --- | --- |
| YAML Schema | 为模块、关卡、角色、怪物、技能、地图、道具、掉落、素材等定义结构模板。 |
| 配置目录管理 | 管理 `control/`、`global/`、`modules/` 三层目录和实体子目录。 |
| 引用校验 | 校验 ID 唯一、引用存在、素材存在、循环引用、难度映射和节点断链。 |
| 发布包生成 | 将后台草稿转换为带 `manifest`、`compatibility`、`indexes/` 的游戏侧只读发布包。 |
| 版本矩阵 | 记录游戏版本、配置版本、模块版本、素材版本和存档兼容说明。 |
| 回滚机制 | 支持回滚到上一稳定发布包，并保留发布记录。 |
| 权限审计 | 记录编辑、审核、发布、回滚人员和变更摘要。 |

## 配置目录建议

```text
game/data/
├── control/
│   ├── home.yaml                              # 首页和公共入口
│   ├── module_index.yaml                      # 模块索引
│   └── publish.yaml                           # 当前发布包元信息
├── global/
│   ├── schemas/                               # YAML schema 或字段模板
│   ├── rules/                                 # 公共规则包
│   ├── characters/                            # 全局角色实体目录
│   ├── monsters/                              # 全局怪物实体目录
│   ├── maps/                                  # 全局地图实体目录
│   ├── skills/                                # 全局技能实体目录
│   ├── items/                                 # 全局道具实体目录
│   └── assets/                                # 公共素材索引
├── modules/
│   └── <module_id>/                           # 单个模块配置与覆盖
└── published/
    └── <publish_version>/                     # 后台生成的游戏侧只读发布包
```

## 发布包最小契约

阶段 0 最小闭环默认至少补齐以下文件：

- `manifest.yaml`
- `compatibility.yaml`
- `indexes/modules.yaml`
- `indexes/entities.yaml`
- `indexes/assets.yaml`

运行时读取顺序默认固定为：

1. 读取 `manifest.yaml`
2. 读取 `compatibility.yaml`
3. 读取三份索引
4. 按需读取 `control/`、`global/`、`modules/` 正文
5. 执行覆盖合并和缓存构建

对应长期架构基线见：

- `doc/1-架构/附录-发布包契约与三态边界.md`
- `doc/1-架构/附录-发布包最小字段契约.md`
- `doc/1-架构/附录-后台发布器与游戏读取器目标分层.md`

## 后台字段建议

| 字段 | 用途 |
| --- | --- |
| `id` | 系统稳定 ID，不随中文名变化。 |
| `display_name` | 中文展示名，可后续接国际化。 |
| `version` | 当前实体配置版本。 |
| `schema_version` | 当前实体结构版本。 |
| `source_path` | 草稿配置来源路径。 |
| `publish_path` | 发布包输出路径。 |
| `asset_refs` | 当前实体引用的素材 ID 列表。 |
| `dependency_refs` | 当前实体依赖的其他实体 ID 列表。 |
| `compatibility_notes` | 发布时写入的兼容说明。 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| AC-0-001 | 新增游戏内容优先只改 YAML 配置，不改 Godot 业务代码。 |
| AC-0-002 | 后台能生成游戏侧只读发布包。 |
| AC-0-002A | 发布包至少包含 `manifest`、`compatibility` 和三份最小索引。 |
| AC-0-003 | 游戏侧能读取发布包并按统一规则合并配置。 |
| AC-0-004 | 配置发布前能校验 ID、引用、素材、难度、关卡节点和存档兼容风险。 |
| AC-0-005 | 中文配置、文案和路径读取保持 UTF-8，不出现乱码。 |
| AC-0-006 | 配置异常时能输出可定位的错误信息，而不是静默失败。 |

## 依赖与输出

| 类型 | 内容 |
| --- | --- |
| 前置依赖 | 无，是所有后续阶段的地基。 |
| 输出给阶段 1-6 | YAML 配置结构、发布包、后台校验、游戏侧加载和稳定 ID 规则。 |
| 输出给阶段 7 | 配置冒烟测试、版本矩阵、回滚验证和存档兼容检查入口。 |
