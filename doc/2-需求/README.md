## 配套资产

- `data/` 用于需求截图、参考图和需求文档引用的外部图片资产。
- 当前 SVG 图表与主文档同目录维护，便于直接按文档主干名检索。
- 同一需求默认保持 `主文档 + .flow.svg + .sequence.svg` 的同主干组合。

## 落点规则

- 需求主文档命名延续 `yyyy-MM-dd_HHmmss_中文主题.md`。
- 需求对应流程图使用同主干名的 `.flow.svg`。
- 需求对应时序图使用同主干名的 `.sequence.svg`。
- 需求截图、参考图和外部生成图片默认放入 `data/`，并在需求主文档中直接引用。
- 同一需求后续迭代默认优先更新原主文档，不再平行新建一份同主题需求说明。
- 当历史需求文档已经分散且用户明确要求重新整理时，允许新建 dated 基线文档承接当前有效结论，并把旧文档降为历史来源。
- 与同一需求绑定的实施总览、实施周期、验收标准和最终验收，统一迁入 `../3-实施/` 与 `../7-验收/` 维护。
