# 道具分层与掉落奖励规范

## 文档说明

- 用途: 统一项目中普通道具、紫色道具、临时金色道具、永久金色道具、金币掉落、掉落表、奖励池和经济权重的长期契约。
- 事实来源: `项目设计.md`、`doc/2-需求/2026-06-23_215445_需求点总览v1.md`、`doc/2-需求/2026-06-08_234500_模块化关卡最高级原则.md`、`doc/1-架构/附录-存档与资产生命周期规范.md`、`doc/1-架构/附录-关卡节点与波次奖励规范.md`、`doc/1-架构/附录-角色固定绑定与选择规范.md`、`doc/1-架构/附录-全配置驱动完善方案.md`
- 适用范围: `game/data/global/items/`、`game/data/global/drops/`、`game/data/modules/<module_id>/items/`、`game/data/modules/<module_id>/drops/`、角色选择外部永久金色道具搭配、关卡奖励配置、Boss 奖励配置、掉落发布校验
- 最近同步时间: 2026-06-28

## 分层总原则

### 1. 道具实体层、掉落表层和关卡奖励层必须分层

- 道具实体层负责“这是什么道具、属于哪种层级、有哪些效果和限制”。
- 掉落表层负责“当前怪物、Boss 或关卡可以掉哪些金币、哪些池子、按什么权重抽取”。
- 关卡奖励层负责“这一关通关后引用哪个奖励表、Boss 额外奖励表和长期奖励概率”。
- 角色选择入口只负责“玩家当前搭配了哪些外部永久金色道具”，不负责定义永久金色道具正文和掉落规则。

### 2. 临时道具与长期外部道具必须严格分层

- 普通、紫色、临时金色道具都属于模块内临时道具。
- 永久金色道具属于长期外部道具，模块结束后不会消失。
- 模块内临时道具池不得混入永久金色道具。
- 长期外部道具池不得承接普通、紫色或临时金色道具。

### 3. 永久金色道具不能通过局内抽奖获得

- 永久金色道具只能通过通关极低概率获得。
- 局内金币抽取池只允许覆盖普通、紫色和临时金色道具。
- Boss 奖励池可以附加长期奖励入口，但仍不得把永久金色写成普通抽奖结果。

### 4. 经济系统必须配置化，不允许把权重和收益曲线写死

- 金币掉落、抽取成本、道具池权重、Boss 奖励池和通关长期奖励概率都必须由配置驱动。
- 经济权重必须可追踪、可校验、可在后台预览。
- 不允许在 Godot 运行时代码中写死某一档颜色权重、固定金币倍率或永久金色概率。

## 目录与载体

### 全局默认目录

```text
game/data/global/
├── items/<item_id>/
│   ├── config.yaml
│   ├── effect.yaml
│   ├── assets/
│   └── overrides/
└── drops/<drop_table_id>/
    ├── config.yaml
    ├── temporary_pools/
    ├── permanent_pools/
    └── boss_bonus_pools/
```

### 模块覆盖目录

```text
game/data/modules/<module_id>/
├── items/<item_id>/
│   ├── config.yaml
│   ├── effect.yaml
│   └── overrides/
└── drops/<drop_table_id>/
    ├── config.yaml
    ├── temporary_pools/
    ├── permanent_pools/
    └── boss_bonus_pools/
```

### 与其他载体的边界

- `game/data/modules/<module_id>/levels/<level_id>/rewards.yaml` 只引用奖励与掉落表，不复制掉落正文。
- `game/data/modules/<module_id>/loadout_rules.yaml` 只约束外部永久金色道具搭配入口，不复制永久金色道具实体正文。
- `game/data/global/characters/<role_id>/` 不直接声明永久金色道具本体，只引用选择输出结果。

## 道具实体分层契约

### `items/<item_id>/config.yaml` 最小字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `item_id` | string | 道具稳定 ID。 |
| `name_zh` | string | 道具中文名。 |
| `item_tier` | string | `normal` / `purple` / `gold_temp` / `gold_permanent`。 |
| `item_scope` | string | `module_temporary` / `external_permanent`。 |
| `item_tags` | string[] | 道具标签，如攻击、防御、经济、召唤、掉落。 |
| `summary_zh` | string | 道具说明。 |
| `effect_ref` | string | 状态效果配置引用。 |
| `icon_asset_id` | string | 图标资源引用。 |
| `stack_rule` | string | 叠加规则摘要；具体叠层语义以状态效果正文为准。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 分层最小约束

- `item_tier=normal` 或 `purple` 时，`item_scope` 必须为 `module_temporary`。
- `item_tier=gold_temp` 时，`item_scope` 必须为 `module_temporary`。
- `item_tier=gold_permanent` 时，`item_scope` 必须为 `external_permanent`。
- `gold_permanent` 道具不得进入局内临时抽取池。
- `status=disabled` 的道具不得出现在正式掉落表或外部搭配列表中。

## 掉落表与奖励池契约

### `drops/<drop_table_id>/config.yaml` 最小字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `drop_table_id` | string | 掉落表稳定 ID。 |
| `module_scope` | string | `global` 或具体 `module_id`。 |
| `coin_rule` | object | 金币掉落规则。 |
| `temporary_item_pool_refs` | string[] | 模块内临时道具池引用。 |
| `permanent_item_pool_refs` | string[] | 长期外部永久金色道具池引用。 |
| `boss_bonus_pool_refs` | string[] | Boss 额外奖励池引用。 |
| `clear_reward_refs` | string[] | 通关结算奖励引用。 |
| `difficulty_override_map` | object | 难度覆盖映射。 |
| `status` | string | `active` / `disabled` / `deprecated`。 |

### 奖励池分类边界

- `temporary_item_pool_refs` 只允许引用普通、紫色、临时金色道具。
- `permanent_item_pool_refs` 只允许引用永久金色道具，并且只能用于通关长期奖励或明确的长期奖励入口。
- `boss_bonus_pool_refs` 负责 Boss 节点的额外奖励层，可包含金币加成、临时道具奖励或长期奖励入口，但必须显式分层。
- `clear_reward_refs` 负责关卡或模块结算结果，不替代掉落表中的即时掉落逻辑。

## 金币与经济权重契约

### `coin_rule` 建议字段

| 字段 | 说明 |
| --- | --- |
| `base_coin_drop` | 基础金币掉落。 |
| `coin_weight` | 金币权重。 |
| `coin_variance` | 掉落波动范围。 |
| `difficulty_coin_ratio` | 难度金币倍率映射。 |
| `boss_coin_bonus_ratio` | Boss 击败额外金币倍率。 |
| `clear_coin_bonus` | 通关结算金币奖励。 |

### 池权重建议字段

| 字段 | 说明 |
| --- | --- |
| `pool_id` | 奖励池稳定 ID。 |
| `pool_type` | `temporary` / `permanent` / `boss_bonus`。 |
| `item_refs` | 奖励池内道具引用。 |
| `weight_map` | 每个道具或颜色层级的权重。 |
| `draw_cost_rule` | 抽取成本或消耗规则。 |
| `max_draw_count` | 当前局或当前结算的最大抽取次数。 |
| `pity_rule` | 保底或保护规则。 |

### 经济权重最小约束

- 金币掉落权重必须能区分普通敌人、精英敌人和 Boss 奖励来源。
- 普通、紫色、临时金色、永久金色四档权重必须显式配置，不允许省略金色分层。
- 永久金色若存在概率字段，必须属于通关长期奖励口径，不得与局内抽取成本规则混写。
- 抽取成本、金币收益和奖励权重变化后，应能通过后台预览中心或校验中心追踪影响。

## 永久金色道具来源限制

### 核心规则

- 永久金色道具只能来自通关极低概率奖励或其他明确长期奖励入口。
- 永久金色道具不能来自背包内金币抽奖。
- 永久金色道具不能来自普通怪即时掉落。
- Boss 奖励池若允许产出永久金色，也必须以“Boss 通关长期奖励”语义显式声明，而不是当作局内随机拾取。

### 与角色搭配入口的边界

- `loadout_rules.yaml` 只约束永久金色道具的携带上限、标签范围和禁止列表。
- 角色选择输出里的 `selected_loadout_item_ids` 只记录本次搭配结果。
- 永久金色道具本体定义、来源概率和长期拥有状态，统一以本文和存档附录为准。

## 模块内临时道具与长期外部道具边界

### 模块内临时道具

- 普通、紫色、临时金色道具属于当前模块生命周期资产。
- 模块内临时道具可通过局内金币抽取、关卡即时奖励或 Boss 临时奖励获得。
- 模块结束后模块内临时道具必须清理。

### 长期外部道具

- 永久金色道具属于长期外部资产。
- 永久金色道具在失败、退出和模块结束后保留。
- 永久金色道具只在角色选择或模块入口搭配阶段进入本次运行态，不直接写回角色正文。

## 发布前最小校验集合

| 校验项 | 通过条件 | 阻断条件 |
| --- | --- | --- |
| 道具分层合法性 | `item_tier` 与 `item_scope` 对应正确 | 永久金色被标成模块临时道具，或反之 |
| 临时池合法性 | `temporary_item_pool_refs` 只含普通、紫色、临时金色 | 临时池混入永久金色 |
| 长期池合法性 | `permanent_item_pool_refs` 只含永久金色 | 长期池混入普通、紫色或临时金色 |
| 永久来源限制 | 永久金色只绑定到长期奖励入口 | 永久金色出现在局内抽奖或普通即时掉落 |
| 金币权重完整性 | 金币规则、抽取成本和池权重可解析 | 权重缺失、抽取成本缺失或引用断链 |
| Boss 奖励边界 | Boss 额外奖励池与关卡通关奖励分层明确 | Boss 奖励、关卡奖励和即时掉落混写 |
| 存档生命周期一致性 | 长期道具与临时道具生命周期和存档规则一致 | 临时道具被持久化为长期资产，或永久金色未进入长期资产 |

## 结论

- 道具系统长期不是“一个抽奖表”，而是“道具实体分层 + 掉落表分层 + 奖励池分层 + 经济权重分层”的组合契约。
- 永久金色道具必须与局内临时抽取彻底分层，保持“只能通关极低概率获得”的长期规则。
- 后续阶段 5 的怪物掉落绑定、阶段 6 的关卡奖励配置、角色外部搭配、后台经济预览和发布校验都应直接复用本文。
