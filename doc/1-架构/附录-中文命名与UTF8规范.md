# 中文命名与 UTF-8 规范

## 目标

本规范用于统一项目中中文命名、UTF-8 编码和稳定 ID 的使用边界。目标是：

- 尽量使用中文，让配置、文档和后台更易读。
- 通过 UTF-8 保证中文可正常存储、读取和发布。
- 同时保留稳定 ID，保证代码、引用和版本兼容不混乱。

## 总原则

1. 玩家可见内容优先中文。
2. 配置内容可以中文，但关键引用必须有稳定 ID。
3. 目录名可以中文，但建议与 ID 保持一致或可映射。
4. 代码层引用优先稳定 ID，不直接依赖中文显示名。
5. 所有中文文件、配置和文案统一使用 UTF-8。

## 适合中文命名的内容

以下内容建议尽量使用中文：

- 模块名
- 关卡名
- 角色显示名
- 怪物显示名
- BOSS 显示名
- 道具显示名
- 技能显示名
- UI 文案
- 按钮文字
- 提示文字
- 后台菜单
- 后台表单标签
- 配置说明文档

## 建议保留稳定 ID 的内容

以下内容不建议只用中文作为唯一标识：

- 实体 ID
- 配置键名
- 引用键
- 文件索引键
- 版本号
- 存档 schema 字段
- 后台路由
- 接口字段

建议采用“双层结构”：

- 中文名：给人看
- 稳定 ID：给系统用

示例：

- 中文名：`庭院模块`
- 稳定 ID：`module_garden`

- 中文名：`小史莱姆`
- 稳定 ID：`enemy_slime`

- 中文名：`火球术`
- 稳定 ID：`skill_fireball`

## 目录命名建议

目录可以中文，但建议遵循以下策略：

- 若目录主要给后台浏览，中文可读性优先
- 若目录需要频繁脚本访问、同步、发布，建议中英混合或稳定 ID 优先
- 最稳妥的方式是目录名使用稳定 ID，目录内再用中文显示名

推荐方案：

- `module_garden/`
- `enemy_slime/`
- `map_graveyard/`

如果确实要中文目录，也应保证：

- 编码统一为 UTF-8
- 路径不出现乱码
- 脚本、打包、同步、Git、后台导出都能稳定识别

## 文件命名建议

### 推荐

- `config.yaml`
- `module.yaml`
- `index.yaml`
- `publish.yaml`

### 可接受

- `庭院模块.yaml`
- `小史莱姆.yaml`
- `墓园地图.yaml`

但如果采用中文文件名，必须满足：

- 全链路 UTF-8
- 工具链不丢失编码
- 脚本和后台都能稳定读写

## 后台显示与存储分离

后台建议分成两层：

- 显示层：中文
- 存储层：稳定 ID

也就是说：

- UI 上显示“庭院模块”
- 数据里存 `module_garden`
- UI 上显示“小史莱姆”
- 数据里存 `enemy_slime`

## 编码规则

### 文档与配置

- 全部使用 UTF-8
- 不允许提交乱码
- 不允许出现中文问号替代字符
- 不允许编码混乱导致的路径失真

### Windows 环境

- 读写中文文件时明确使用 UTF-8
- 批量修改后要检查 diff，确认没有乱码或无关换行变化

## 引用规则

- 中文名只用于展示
- 稳定 ID 用于关联
- 目录和文件名如果是中文，后台必须有映射能力
- 任何跨模块、跨版本、跨存档的引用都优先使用稳定 ID

## 推荐落地方式

最推荐的组合是：

- 中文显示名
- 稳定 ID
- UTF-8 编码
- 目录名与 ID 尽量对齐

这样可以同时满足：

- 人类可读
- 工具可用
- 跨平台稳定
- 后台可选
- 游戏可读

## 结论

本项目可以尽量中文命名，但必须遵守：

- 中文负责可读性
- ID 负责稳定性
- UTF-8 负责编码安全

如果两者冲突，优先保证稳定 ID 和工具链兼容，再保证中文展示体验。
